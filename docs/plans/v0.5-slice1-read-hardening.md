# Plan: v0.5 Slice 1 — read-hardening remainder (S1 + S3 + C3-pagination)
Status: approved · Date: 2026-06-26 · Slug: `v0.5-slice1-read-hardening`
Defaults approved: page size 500 · default --timeout 30s · --timeout 0 = no deadline (signal-only)
Issue: #19 (slice) · resolves #15 (S1), #17 (S3), #14-part (C3 pagination)
Parent: [`v0.5-admission-webhook.md`](v0.5-admission-webhook.md) — slice 1

## Problem
Slice 1 (read hardening) makes the v0.4 one-shot live reader honest on a real cluster.
Its first item, **C1** (tolerate `ErrGroupDiscoveryFailed`), shipped in PR #30. Three
items remain on the same read path:

- **S1 — no timeout/cancellation.** `cmd/krsm/main.go:260` uses `context.Background()`; a
  hung API server hangs `krsm check` forever, and Ctrl-C / SIGTERM cannot cleanly abort.
- **S3 — multi-version resources are double-listed.** `Reader.State` selects list targets
  from `ServerGroupsAndResources()`, which returns *every served version*; `selectTargets`
  dedups only by GVR, so a resource served at two versions (e.g. `HorizontalPodAutoscaler`
  at `autoscaling/v1` **and** `/v2`) is listed twice. The uid-keyed closure walk dedups the
  *verdict*, so this is waste + a footgun, not a wrong answer — fix by listing **preferred
  versions only**.
- **C3-pagination — unbounded list.** `reader.go:113` does `List(ctx, metav1.ListOptions{})`
  with no `Limit`: one check pulls every object of a kind in a single call — a latency/OOM
  hazard on a large cluster. (The Secret-`data` half of C3 — a metadata-only informer — is
  slice 3, not here.)

## Goals / Non-goals
- **Goal:** `krsm check` against a live cluster honours a default `--timeout` and cancels
  on SIGINT/SIGTERM, failing **closed** on deadline/cancel (S1).
- **Goal:** the reader lists each resource at its **preferred version only**, so a
  multi-version built-in is listed once; scope discovery still reads all versions for the
  namespaced flag (S3).
- **Goal:** every `List` is **paginated** (`Limit` + `Continue`), accumulating pages, with
  a fail-closed error on any page (C3-pagination).
- **Non-goal:** the metadata-only Secret informer / dropping Secret `data` (slice 3, C3
  proper).
- **Non-goal:** any informer/`state` code, staleness guard (slices 3–4).
- **Non-goal:** any change to `closure`/`scope`, `closure.Safe`, or goldens 01–27.
- **Non-goal:** changing C1's discovery tolerance (reused, not modified).

## Current state
- `cmd/krsm/main.go`
  - `runCheck` (155) builds a `flag.FlagSet` with `--plain/--mode/--context/--kubeconfig/-n`;
    no `--timeout`. `liveOpts` (215) carries the flags into the live path.
  - `runCheckLive` (237) builds the reader, then `ctx := context.Background()` (260), then
    `ResolveKind(ctx)` and `State(ctx)`. `main` (116) maps a non-`errBlocked` error → exit 1.
- `internal/cluster/reader.go`
  - `State` (98): `lists ← serverResources(disc)` (C1-tolerant `ServerGroupsAndResources`);
    `scope ← scopeFromLists(lists)`; `targets ← selectTargets(lists)`; then per-target
    `r.dyn.Resource(t).List(ctx, metav1.ListOptions{})` (113), no `Limit`.
  - `serverResources` / `closureRelevantFailures` (268+): the C1 partial-discovery wrapper.
  - `selectTargets` (164) dedups by GVR; `scopeFromLists` (236) needs the per-version
    `Namespaced` flag. `DiscoveryInterface` exposes `ServerPreferredResources()`.
- Tests: `reader_test.go` drives a `discoveryfake.FakeDiscovery{Resources: resourceLists()}`
  + `dynamicfake`. **Note:** `FakeDiscovery.ServerPreferredResources()` returns `(nil,nil)`,
  so S3 needs the shared test discovery to also serve preferred resources (default it to the
  same lists so existing tests are unaffected). `TestReaderUsesOnlyReadVerbs` /
  `TestPackageInvokesNoWriteVerbs` are the read-only guards that must stay green.

## Approach
One branch, three vertical slices in order S1 → S3 → C3, each TDD (red → green) and
`make check` green before the next. Combined per maintainer decision; one design doc.

- **S1:** add `--timeout` (default `30s`) to the check FlagSet; carry it in `liveOpts`. In
  `runCheckLive`, derive the context from `signal.NotifyContext(Background, os.Interrupt,
  SIGTERM)` then `context.WithTimeout(ctx, o.timeout)` (defer both cancels). The existing
  `State(ctx)`/`ResolveKind(ctx)` already take ctx; a deadline/cancel surfaces as a
  list/discovery error → the existing fail-closed `check: cluster read failed…` path. A
  `--timeout 0` means no deadline (signal-only). Hermetic test: a tiny timeout + a fake
  reader that blocks → `runCheckLive` returns a fail-closed error (exit 1), no partial
  verdict.
- **S3:** add a C1-style tolerant `serverPreferredResources(disc)` wrapping
  `disc.ServerPreferredResources()` (same `ErrGroupDiscoveryFailed` handling). In `State`,
  keep `serverResources` for `scopeFromLists` (all versions → namespaced flag) but feed
  `selectTargets` the **preferred** lists. Result: one GVR per resource → listed once.
  Hermetic test: a fake whose `ServerGroupsAndResources` serves HPA at `autoscaling/v1`+`/v2`
  but `ServerPreferredResources` serves only `/v2` → exactly one HPA `List`, at `v2`.
- **C3:** replace the single `List` with a `listAll(ctx, gvr)` helper that loops on
  `ListOptions{Limit: N, Continue: token}` until `Continue == ""`, appending `Items`; any
  page error fails closed. `N` a small const (e.g. 500). Hermetic test: a dynamic `list`
  reactor returning a non-empty `Continue` on page 1 and the rest on page 2 → all items
  present, ≥2 `list` actions observed.

## Steps (vertical slices)
1. [x] **S1 — timeout + signal cancel.** `--timeout` flag, `liveOpts.timeout`,
   `signal.NotifyContext` + `WithTimeout` in `runCheckLive`; fail-closed on deadline.
   *Demo: `krsm check --context … --timeout 1ms <verb> <target>` exits 1 fail-closed.*
2. [x] **S3 — preferred-version listing.** `serverPreferredResources` (C1-tolerant);
   `State` lists preferred GVRs, scopes from all. *Demo: a two-version HPA is listed once.*
3. [x] **C3 — paginated list.** `listAll` with `Limit`/`Continue`. *Demo: a multi-page
   fake list returns every item across ≥2 calls.*

## Risks & unknowns
- **Test doubles.** `FakeDiscovery.ServerPreferredResources` returns nil → must extend the
  shared test discovery (default preferred = all) so existing reader tests stay green; the
  multi-version case uses an explicit override. The dynamic fake ignores `Limit/Continue`,
  so the pagination test drives a `list` reactor that honours `Continue`.
- **Read-only invariant.** `listAll` still uses only `Resource(...).List`; the no-write
  guards (`TestReaderUsesOnlyReadVerbs`, `TestPackageInvokesNoWriteVerbs`) must stay green.
- **Signal handling testability.** `signal.NotifyContext` is hard to unit-test directly;
  S1's *timeout* branch is the testable fail-closed path; signal cancel is argued + covered
  by the deadline test sharing the same code path. Keep `main` thin.
- **`--timeout 0` semantics.** Document as "no deadline" (signal-only) to avoid an instant
  deadline-exceeded.

## Definition of done
- `--timeout` defaults to 30s, cancels on SIGINT/SIGTERM, fails closed on deadline (S1).
- Multi-version resources listed once at preferred version; scope still all-version (S3).
- All live `List`s paginated; a multi-page result is fully assembled; page error → closed.
- `internal/cluster` read-only guards green; `closure`/`scope`/goldens untouched.
- `make check` green. Lands as one branch/PR (#19), base `release/v0.5.0` (C1-inclusive).
