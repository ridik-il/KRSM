# Plan: v0.5 state path — indexed informer State + staleness guard (C2 + C3) + CLI group-aware target (S2)
Status: approved · Date: 2026-06-27 · Slug: `v0.5-state-c2-c3-s2`
Issue: #21 (slice 3) + #22 (slice 4) · resolves #14 (C3), #13 (C2), #16 (S2)
Parent: [`v0.5-admission-webhook.md`](v0.5-admission-webhook.md) — slices 3–4 (+ the CLI half of S2)
Decisions approved: one branch/PR for all three steps · `project` stays in `internal/cluster` (imported by `state/`) · `FreshGet` is metadata-typed for Secret/ConfigMap (never pulls `data`) · staleness signal is `resourceVersion` primary (`generation` a spec-change hint) · participating CRDs use full informers

## Problem
Slice 1 (read hardening) and slice 2 (shared `project` projection) shipped. Three v0.4
real-world gaps remain on the live-read path, and all three are the natural product of the
hot-path the webhook needs anyway — an **informer-backed, indexed** `closure.State` (ADR-0004,
DESIGN §4):

- **C3 (#14) — the one-shot read pulls every Secret's `data` into memory.** Slice 1 added
  `Limit` pagination, but the one-shot reader still `List`s full objects, so `Secret.data` /
  `ConfigMap.data` are fetched and held — an OOM/latency hazard *and* a secret-exposure
  surface for a security control that never reads `data` (the four relations need only
  metadata: labels / ownerReferences / finalizers / spec refs).
- **C2 (#13) — the read is not a consistent snapshot.** `Reader.State` issues a separate
  `List` per GVR, each at its own resourceVersion (`internal/cluster/reader.go`); a neighbour
  created/deleted mid-read yields a closure over torn state → a **false ALLOW (missed
  collateral)**, the one error class a safety gate must not have.
- **S2 (#16) — group-blind target resolution.** `Reader.ResolveKind` returns the first Kind
  match and discards the API group; same-Kind-across-groups (CRD-heavy clusters) resolves to
  whichever group discovery enumerates first → wrong target, wrong closure.

DESIGN §4 already decided the fix is informer-fed inverted indexes costing `O(c·d)` (blast
radius), not `O(c·n)` (cluster size). A single consistent watch-fed cache fixes C2's torn
read; metadata-only informers fix C3's secret exposure; both live in a new `state/` package.
S2's *full* fix is the webhook (the `AdmissionReview` carries the real GVK + uid — slice 5);
this branch closes S2 on the **CLI** path with a group qualifier and a fail-closed
disambiguation, so #16 is honestly resolved now and moot later.

## Goals / Non-goals
- **Goal (slice 3, C3 + C2-part):** a new **`state/` package** — an informer-backed,
  incrementally **indexed** second `closure.State` (beside `closure.NewScanState`), serving
  all nine methods from four inverted indexes in `O(c·d)`. **Metadata-only**
  (`PartialObjectMetadata`) informers for Secrets *and* ConfigMaps so `data` is never fetched
  or cached. `Start` + `WaitForCacheSync`; serving before sync **fails closed**. Reuses the
  slice-2 `project` projection so it reproduces goldens 01–27 (corpus parity oracle).
- **Goal (slice 4, C2):** a per-request **staleness guard** — `FreshGet` a bounded on-demand
  GET over the `O(d)` closure neighbourhood; compare the request target's
  `resourceVersion`/`generation` to the cached copy; on drift reconcile via `FreshGet` (never
  a re-list); **fail closed on unresolved drift** with a distinct reason (ADR-0004).
- **Goal (S2, #16):** **group-aware** CLI target resolution — accept `Kind.group/name` (e.g.
  `Deployment.apps/web`); when a bare Kind is served by multiple groups and none is given,
  **fail closed** asking the operator to disambiguate (no silent first-match).
- **Goal (security, DESIGN §9):** Secret/ConfigMap `data` structurally absent from cache;
  read-only get/list/watch only (no write, no wildcard) enforced by a package source-guard
  test; a fail-closed taxonomy with distinct, credential-free reasons.
- **Non-goal:** the `webhook/` server (slice 5), progressive scope channel (slice 6),
  deployment / Helm / RBAC *manifests* (slice 7). RBAC here is only the **documented**
  read-only contract the package assumes; the manifest lands in slice 7.
- **Non-goal:** any change to `closure.Safe`'s signature, the closure walk, `scope/`, the
  derived-scope/mode code, `closure.Ref.human` (group-agnostic by design), or goldens 01–27.
  The indexed `State` plugs into the **unchanged** interface seam.
- **Non-goal:** changing the one-shot `internal/cluster.Reader` behaviour (it stays the CLI
  read path; the informer `Provider` is the new, separately-tested second implementation —
  there is no webhook in this branch to consume it yet, so its consumer is its tests).
- **Non-goal:** `closure`/`scope` gaining non-stdlib deps; `state/` sits **outside** that
  tree (archguard stays green).

## Approach
Build the new `state/` package bottom-up and prove it hermetically against the corpus parity
oracle **before** any cluster — the index must reproduce `NewScanState`'s closure/escaping set
exactly. The staleness guard and the CLI S2 fix are independent, small slices around it.

### Decisions taken
1. **A second `closure.State`, not a rewrite of the one-shot reader.** `state.Provider`
   implements the existing interface; the engine, scope, and goldens are untouched. The
   one-shot reader stays the CLI path; the `Provider`'s consumer is the webhook (slice 5),
   so in this branch its only consumer is its test suite + the parity oracle.
2. **Four inverted indexes under one `sync.RWMutex`** (owner `uid→[]childRef`; namespace
   `ns→[]ref`; selector inverted-index `ns→[]selectorOwnerRef` + per-owner `LabelSelector`;
   reverse cross-ref `referentKey→[]consumerRef`), maintained by informer add/update/delete
   handlers; the nine reads take the read lock and return `[]closure.Ref`. Indexes store only
   `project`-shaped `closure.Object`s, never raw API objects.
3. **Metadata-only informers for Secrets + ConfigMaps (C3 security core).** A
   `metadatainformer.SharedInformerFactory` (`PartialObjectMetadata`) for kinds whose only
   relational role is being *referenced* (Secret, ConfigMap) so `data` is never fetched or
   cached; full informers for kinds whose labels/spec the relations read (Pods, workloads,
   Services, PDBs, NetworkPolicies, HPAs, Namespaces, PVCs, participating CRDs).
   `PartialObjectMetadata` still carries labels/ownerReferences/finalizers — all the relations
   read for those kinds.
4. **Preferred-version GVRs from the slice-1 selection (S3 reuse).** One informer per
   preferred GVR via the hardened `serverPreferredResources`/`selectTargets`; reuse the
   C1 partial-discovery tolerance at factory startup so a missing aggregated group does not
   crash informer start.
5. **`WaitForCacheSync` gates serving (fail-closed).** `New` does not block; `Start` launches
   informers; `WaitForCacheSync` must return true before any verdict. An unsynced cache is an
   unknown closure → fail closed (the caller maps this to a deny; the package exposes the
   sync state, it does not itself render a verdict).
6. **Staleness guard = drift-triggered bounded `FreshGet`, not always-GET (ADR-0004 Q3).**
   Primary read is the index. Given the authoritative request object, compare its
   `resourceVersion`/`generation` to the cached copy; on drift, `FreshGet` the `O(d)`
   neighbourhood (never a re-list); unresolved drift **fails closed** with a distinct reason.
   Built and unit-tested hermetically (fake client returning newer-than-cache) since the
   webhook that supplies the request object is slice 5.
7. **S2 on the CLI: group qualifier + fail-closed disambiguation, no engine change.** Parse
   `Kind.group/name` (`Deployment.apps/web`); `ResolveKind` becomes group-aware — a bare Kind
   served by >1 group with no qualifier is a fail-closed error listing the candidates, not a
   silent first-match. `Ref.human` stays group-agnostic (the goldens depend on it); the
   webhook's uid match is the complete fix, documented as such.
8. **Read-only by construction + source-guard.** `state/` uses only get/list/watch; a
   `TestPackageInvokesNoWriteVerbs`-style guard (mirroring `internal/cluster`) forbids any
   write verb. Reasons/logs never echo `rest.Config`, tokens, or object `data`.

### Dependency / supply-chain position
Promote `k8s.io/api` to a direct dep; add `client-go` `metadatainformer`/`dynamicinformer` +
`tools/cache` (already in tree). `state/` deps must not leak into `closure`/`scope` (archguard
+ `go list -deps ./closure/... ./scope/...` stays k8s-free). Resolve any `govulncheck` advisory
by bump, never suppress (v0.4 precedent).

## Steps (vertical slices)
Development is **strictly test-driven**, via the `design-check-tdd → tdd` pipeline: for each
slice, first write the failing test(s) that encode the wanted behaviour below (red), then
implement to green, then refactor — never code ahead of a test. The named **Wanted behaviour**
list per slice is the concrete set of acceptance assertions that `design-check-tdd` turns into
the ordered failing-test list; each bullet is one test seed.

1. [ ] **Indexed informer `state.Provider` implementing `closure.State` (C3 + C2-part).** New
   `state/` package: `New`/`Start`/`WaitForCacheSync`; the four inverted indexes + event
   handlers consuming the shared `project`; full + metadata-only informer split; the nine
   `State` methods over the indexes; read-only source-guard. (#14, #13-part)
   - **Wanted behaviour (test seeds):**
     - `state.Provider` satisfies `closure.State` (`var _ closure.State = (*Provider)(nil)`).
     - The indexed `Provider` reproduces goldens 01–27's closure/escaping set — the **corpus
       parity oracle** (same verdict + same escaping refs as `NewScanState`).
     - `Get`, `OwnedChildren`, `NamespaceContents`, `PodsSelectedBy`, `PodsMatching`,
       `SelectorsTargeting`, `SelectorsMatchingLabels`, `Consumers`, `ControllersTargeting`
       each return the **same `Ref` set as `NewScanState`** on the corpus fixtures (order-
       insensitive).
     - A cached Secret object carries `labels`/`ownerReferences`/`finalizers` but **no
       `data`** (metadata-only); same for ConfigMap.
     - A **metadata-only Pod still binds a Service selector** (`PartialObjectMetadata` labels).
     - A multi-version resource is informed **once** (preferred version only).
     - **Serving before `WaitForCacheSync` returns true fails closed** (the not-synced state is
       observable and distinct, never a silent empty closure).
     - **No write verb is invoked** by the package (source-guard test, mirroring
       `internal/cluster.TestPackageInvokesNoWriteVerbs`).
     - Concurrent handler-writes vs `State`-reads are `go test -race` clean.
2. [ ] **Staleness guard + bounded neighbourhood fallback (C2).** `Provider.FreshGet`; the
   resourceVersion/generation drift check + `O(d)` reconciliation; fail-closed-on-unresolved
   with a distinct, credential-free reason. (#13)
   - **Wanted behaviour (test seeds):**
     - A request target **newer than cache** (higher `resourceVersion`) triggers a bounded
       `FreshGet` over the `O(d)` neighbourhood — **not a re-list** (assert no `list` action).
     - When the cache already matches the request `resourceVersion`, **no `FreshGet`** fires
       (the hot path stays read-free).
     - **Unresolved drift fails closed** with the distinct reason `"could not confirm current
       state"` — never an admit over stale state.
     - `FreshGet` for a metadata-only kind (Secret/ConfigMap) **does not fetch `data`**.
     - The drift reason and any log line **never echo** `rest.Config`, tokens, or object `data`.
3. [ ] **CLI group-aware target resolution (S2).** `Kind.group/name` parsing in the target
   split; group-aware `ResolveKind` with fail-closed disambiguation on ambiguous bare Kinds.
   (#16)
   - **Wanted behaviour (test seeds):**
     - `Kind.group/name` (e.g. `Deployment.apps/web`) **resolves to the qualified group's**
       GVR/Kind.
     - A **bare Kind served by >1 group with no qualifier errors fail-closed**, listing the
       candidate groups — never a silent first-match guess.
     - An **unambiguous bare Kind** (served by exactly one group) still resolves as today
       (no regression to existing CLI tests).
     - A `Kind.group/name` whose group does not serve that Kind errors fail-closed.

## Risks & unknowns
- **Index/loader parity (highest).** The index must reproduce `project`'s four relations and
  `BuildObjects`'s cross-ref-uid resolution exactly, or the live verdict diverges from the
  goldens. Mitigation: the corpus parity oracle in slice 1 gates it before any cluster; reuse
  `project` verbatim (no re-derived field paths). Note: `BuildObjects` fills cross-ref uids in
  a whole-set post-pass; the index has no whole-set moment, so it must rely on the engine's
  Kind/ns/name cross-ref fallback (`crossRefMatches`) — pin that this yields the same closure.
- **Cache staleness vs the watch-write race (ADR-0004).** Bounded by slice 2, not eliminated
  (K8s has no multi-resource snapshot primitive). Mitigation: drift-triggered `FreshGet` +
  fail-closed; document the residual window. The real request object arrives only with the
  webhook (slice 5) — this branch tests the mechanism, not the end-to-end admission race.
- **Metadata-vs-full informer split correctness (C3).** A kind needing labels for selector
  binding but fetched metadata-only would silently break selector binding.
  `PartialObjectMetadata` *does* carry labels, so Secrets/ConfigMaps (referenced-only) are
  safe metadata-only; Pods (selector-relevant) stay full. Pin with a metadata-Pod-binds-a-
  Service test.
- **Informer startup on a cluster missing a group.** Reuse the C1 partial-discovery tolerance
  so a missing/flaky aggregated group does not crash `Start`.
- **`-race` correctness.** Concurrent informer-handler writes vs `State`-method reads — the
  single `RWMutex` and slice-copy returns must be `go test -race` clean.
- **S2 scope.** The CLI fix does not change `Ref.human`; two same-Kind objects in different
  groups in the *same* ns/name still share a human key in the engine — acceptable because the
  CLI now resolves the right GVR and the webhook (uid) is the full fix; documented.

## Definition of done
- `state.Provider` implements `closure.State`; the indexed provider reproduces goldens 01–27's
  closure/escaping set (parity oracle). (#14, #13)
- Cached Secret/ConfigMap objects carry labels/ownerReferences/finalizers but **no `data`**;
  a metadata-only Pod still binds a Service selector. (#14)
- Serving before `WaitForCacheSync` fails closed; the fail-closed reasons are distinct and
  credential-free; the `state/` no-write source-guard is green. (security)
- Drift against a newer-than-cache object triggers a bounded `FreshGet`; unresolved drift
  fails closed with the distinct reason. (#13)
- `krsm check` accepts `Kind.group/name`; an ambiguous bare Kind fails closed listing
  candidates; unambiguous bare Kinds unchanged. (#16)
- `closure`/`scope` stay stdlib-only (archguard green); `state/` outside both trees;
  `closure.Safe`/walk/`Ref.human`/goldens 01–27 unchanged.
- `make check` green (`gofmt`, `go vet`, `golangci-lint`, `staticcheck`, `go test -race`); CI
  green incl. `govulncheck`/`dependency-review`/CodeQL/Scorecard. Lands as one branch/PR, base
  `release/v0.5.0`.
