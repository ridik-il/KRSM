# Plan: v0.2-cli-check
Status: approved · Date: 2026-06-20 · Slug: `v0.2-cli-check`

## Problem
v0.2's done-when in [ROADMAP §v0.2](../ROADMAP.md) has two clauses: *"N scenarios pass; **each
doubles as a runnable 'static admission misses this, KRSM catches it' demo.**"* The first is met
(12 golden scenarios green under `go test -race`); the second is **not**. The `krsm` CLI
(`cmd/krsm/main.go`) only handles `version`/`help` and never touches the `closure` engine, and the
README's `ACTION / SCOPE / CLOSURE / VERDICT ❌ BLOCK` block (README:50-59) is aspirational — no
command produces it. This slice makes the corpus runnable and the README real.

## Goals / Non-goals
- **Goal:** a `krsm check <scenario-dir>` command that loads a scenario (`cluster.yaml`,
  `request.yaml`, `scope.yaml`), runs `closure.Safe`, prints the closure / escaping set / verdict,
  and sets a scriptable exit code.
- **Goal:** every one of the 12 corpus scenarios becomes a one-command demo; the README demo block
  becomes reproducible output.
- **Goal:** the scenario loader becomes reusable (callable from non-test code) — today it lives in
  `closure/loader_test.go`.
- **Non-goal:** live-cluster reads / `client-go` (that's v0.4 — `krsm check <cluster>`).
- **Non-goal:** the `TaskContract` scope language (v0.3); scope input stays the flat `scope.yaml`.
- **Non-goal:** changing the `closure` package's **public API** — it must stay stdlib-only / YAML-free
  (DESIGN §7). The loader is a dev/demo concern, not the SDK.
- **Non-goal:** a `--verify`/expected-comparison mode (that's what `TestScenarios` already does) and
  JSON output — both are easy later adds, deferred to keep the slice tight.

## Current state
- `cmd/krsm/main.go:40` — `run(args []string, out io.Writer) error` dispatches `version`/`help` via a
  `switch`; clean and unit-tested (`cmd/krsm/main_test.go`). Adding a subcommand is local.
- `closure/` — public SDK: `Safe(State, Action, []ScopeRef) Decision`, `Closure`, `NewScanState`.
  `Decision{Verdict, Reason, Closure, Escaping, External}`. No YAML, no client-go in its API.
- `closure/loader_test.go` — **test-only**, `package closure`: `parseCluster/parseAction/parseScope`,
  the `raw*` structs, uid synthesis, and `sigs.k8s.io/yaml` live here. `loadScenario(dir)` reads the
  four files. It uses only **exported** `closure` constructors (`NewScanState`, `Object`, `Ref`, …),
  so it can move out of `package closure` without needing internals.
- `closure/testdata/scenarios/<NN-name>/{cluster,request,scope,expected}.yaml` — the 12 scenarios.
- `sigs.k8s.io/yaml v1.4.0` is already a `go.mod` require (used only by tests today).

## Approach
Extract the scenario loader into a reusable **`internal/scenario`** package that imports `closure`
and depends on `sigs.k8s.io/yaml`, then add a `check` subcommand to the existing `run` dispatcher
that calls it. `internal/` is the right home: external embedders use `closure`/`scope` directly and
bring their own state (or client-go in v0.4); the YAML scenario loader is this module's own
dev/demo tool. This keeps the SDK pure (the constraint that ejected the parity harness earlier) while
giving both the CLI and the golden test one loader instead of two copies.

Output mirrors the README shape (`ACTION / SCOPE / CLOSURE / VERDICT`) and the verdict drives the
exit code so the demo is scriptable (`BLOCK` → non-zero). The richer per-line annotations in the
README mock (`(live replica)`) are beyond what the engine returns; the demo prints the engine's real
refs and reasons — fidelity-to-mock is a design-stage call, not a goal.

## Steps (vertical slices)
1. [ ] **Tracer bullet — `krsm check <dir>` end to end.** Extract just enough loader into
   `internal/scenario` to load a scenario dir, add the `check` case to `run`, print a single
   `VERDICT <Allow|Warn|Block> — <reason>` line, and set the exit code (Block → non-zero, Allow/Warn
   → 0). Demo: `krsm check closure/testdata/scenarios/01-memory-pressure-cascade` prints `BLOCK` and
   exits non-zero. Test via `run(args, out)` against a scenario dir.
2. [ ] **Full demo output.** Print the README-shaped block: `ACTION` (verb + target), `SCOPE`
   (clauses), `CLOSURE` (sorted refs), the escaping list under a `BLOCK`, and the `External` list
   under a `WARN`. Demo: output for scenario 01 matches the README block; scenario 09 shows the WARN
   + external path.
3. [ ] **Polish + retire the overclaim.** Add `check` to the usage banner and a `krsm check --help`;
   update the README to present the demo as real (with the actual command); add a CHANGELOG entry.
   Optionally fold the golden `TestScenarios` onto the extracted loader so there's a single loader
   (or leave a thin shim) — decide in design.

## Risks & unknowns
- **Loader move must not break `TestScenarios` or the `closure` public API.** The golden test and its
  `testdata/` are package-relative to `closure/`; if the loader moves, decide whether the golden test
  stays in `package closure` (importing `internal/scenario`) or moves with it (and how it then finds
  `testdata/`). Resolve in design — keep `closure/` API YAML-free either way.
- **Exit-code semantics for `Warn`.** Block must be non-zero; is `Warn` a 0 (advisory) or a distinct
  non-zero? Affects scriptability. (Leaning: Warn → 0, distinct stderr note.)
- **Offline `go mod tidy`** already can't complete fully offline (yaml's test deps uncached) — known,
  harmless for build/test; moving yaml into non-test code keeps it a normal require.

## Definition of done
- `krsm check <dir>` prints the correct closure/escaping/verdict for **all 12** scenarios, with
  Block → non-zero exit and Allow/Warn → 0.
- The README demo block is reproducible by running the documented command (no longer aspirational).
- `closure` package public API unchanged and still YAML-free; one shared scenario loader.
- Full gate green: `gofmt`, `go vet`, `staticcheck`, `golangci-lint`, `go test -race ./...`.
- ROADMAP v0.2 "each scenario doubles as a runnable demo" is satisfied.

## Open questions — RESOLVED (approved 2026-06-20)
1. **Loader home → `internal/scenario`.**
2. **Golden `TestScenarios` stays in `package closure`** and imports `internal/scenario`, so
   `testdata/` remains package-relative and fixtures don't move.
3. **`check` input is directory-only** for v0.2; `--cluster/--request/--scope` flags wait for v0.4.
4. **`Warn` → exit 0** (advisory, with a distinct stderr note); **`Block` → non-zero**.
5. **Output prints the engine's real refs + `reason`**; drop the editorial `(live replica)`
   annotations (not engine-produced data).
