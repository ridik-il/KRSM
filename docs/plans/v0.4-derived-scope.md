# Plan: v0.4-derived-scope
Status: done · Date: 2026-06-22 · Slug: `v0.4-derived-scope`

## Problem
KRSM is valueless until a user authors a `TaskContract` (no contract → no `scope(T)` →
no verdict). ADR-0011 fixes this with **progressive, derived scope**: `scope(T)` can be
*synthesized* from the action target + live topology instead of always declared, while
`C(S,A) ⊆ scope(T)` is preserved (only the *provenance* of `scope(T)` changes).

This plan covers the first cut of that work — the **scope synthesizer** and the two
scope dimensions it needs — so that `krsm check <dir>` with **no** `scope.yaml` and **no**
`taskcontract.yaml` derives a conservative Level-0 scope from the request's target and
returns a verdict, and a `--mode audit|enforce` switch governs whether a derived-scope
escape is a `Warn` or a `Block`. It is demoable **offline on the corpus** today; the live
informer side of v0.4 and the webhook (v0.5) consume the same synthesizer later.

## Goals / Non-goals
- **Goal:** an `ownership` scope dimension (DESIGN §6) — root + transitive owned descendants — evaluated state-dependently in the engine, reusing the *same* owner→child walk `Closure` uses so scope-ownership and closure-ownership agree by construction.
- **Goal:** a `namespace` scope dimension — "everything in namespace N" — a pure-`Ref` match (no state).
- **Goal:** a pure, stdlib-only **synthesizer** in `scope` that derives a Level-0 `ScopePredicate` from a target: a single ownership clause rooted at the target. (Not OR'd with a namespace clause — under the union semantics of `C ⊆ scope` that would re-admit the same-namespace collateral the tree is meant to flag, and it is redundant since cross-namespace members are already outside the tree.)
- **Goal:** the loader/CLI derive Level-0 scope when neither `scope.yaml` nor `taskcontract.yaml` is present, so the corpus runs with zero declared scope.
- **Goal:** an **audit/enforce** mode that maps a derived-scope escape `Block → Warn` in audit (the install default), applied *above* `closure.Safe`.
- **Non-goal:** live client-go informers / `kind` snapshotting (rest of v0.4, separate plan).
- **Non-goal:** the admission webhook, `namespaceSelector`/`objectSelector` gating, enforce-as-deny-pre-persistence (v0.5).
- **Non-goal:** the `krsm.io/target` annotation (Level 1) and templates (Level 2, v0.6).
- **Non-goal:** the `reference` scope dimension and inexpressible-scope gap signalling.
- **Non-goal:** any change to `closure.Safe`'s signature or the closure walk.

## Current state
- `closure/model.go` — `ScopeClause{Dim, GVK, Namespace, Name, Selector}` with `DimResource`/`DimSelector`; `ResourceClause`/`SelectorClause` constructors; `ScopeClause.Validate()`.
- `closure/safe.go` — `Safe(State, Action, []ScopeClause)`; `matchScope(r, scope, labels labelsOf)` with an explicit dimension switch (resource/selector/default-skip); `labelsOf func(Ref)(map[string]string,bool)` adapter built by `Safe` over `s.Get` (`safe.go:33`).
- `closure/state.go` — `State` interface; `OwnedChildren(r) []Ref` (`state.go:73`) already does owner→child by uid; `NamespaceContents(ns)`.
- `closure/closure.go` — `Closure` cascades owner→child via `s.OwnedChildren` (`closure.go:40`) with a visited-set guard (`|C| ≤ |R|`, terminating).
- `scope/scope.go` — `TaskContract` → `Compile` → `ScopePredicate{Clauses, MaxSeverity}`; `compileClause` owns the dim→clause mapping; stdlib-only (`internal/archguard` `TestScopeIsStdlibOnly`).
- `internal/scenario/scenario.go` — `loadScope` prefers `taskcontract.yaml` → `parseScope` fallback; `cmd/krsm` renders the SCOPE line.
- Golden corpus: `01-memory-pressure-cascade` is the **intra-namespace** cascade (the Level-0 ownership default must catch it; a namespace-only default would not).

## Approach
A thin layer **above** the unchanged engine, plus two new dimensions **inside** it.

1. **Two new dimensions in `closure`.**
   - `DimNamespace` — gates on `Namespace` (and GVK when set); matches every candidate in that namespace. Pure `Ref`, no state. (`Name`/`Selector` must be empty — `Validate`.)
   - `DimOwnership` — carries a `Root Ref` (and is "include descendants"); a candidate `r` is in scope iff `r` is the root or lies in the root's transitive owned subtree. **State-dependent**, so it forces the `matchScope` seam to widen (below).
   - Add `NamespaceClause(gvk, ns)` and `OwnershipClause(root Ref)` constructors and extend `Validate` (each dimension carries only its own fields).
2. **Widen the `matchScope` seam — the central decision.** `labelsOf` (labels-by-ref) cannot answer ownership-subtree membership. Replace the bare `labelsOf` parameter with a **narrow resolver interface** the `Safe` caller satisfies from the `State` it already holds — e.g. labels-by-ref **and** ownership-subtree membership — rather than accumulating a second function parameter (the smell the v0.3 review flagged). Ownership membership is computed by the **same** owner→child BFS `Closure` uses (`s.OwnedChildren` from the root, visited-set guard), so the two notions of "owned" can never diverge, and the walk stays `≤ |R|`/terminating. `matchScope` stays unexported; `Safe`'s public signature is unchanged.
3. **A pure synthesizer in `scope`.** `Derive(target closure.Ref) ScopePredicate` (Level 0) returns `{Clauses: [OwnershipClause(root=target)]}` — the ownership tree alone (no namespace clause; see Goals). State-free: the state-dependence lives in the engine's `matchScope`, exactly as for selector — so the synthesizer stays stdlib-only and trivially testable. Provenance is recorded for the verdict reason.
4. **Loader/CLI wiring.** When a scenario directory has neither `scope.yaml` nor `taskcontract.yaml`, `loadScope` calls `scope.Derive(action.Target)` and uses its clauses. The CLI renders ownership/namespace clauses readably (e.g. `owns:Deployment/prod/web`, `ns:prod/*`) and reports provenance (`scope: derived (ownership-tree)`).
5. **Audit/enforce mode above `Safe`.** A `scope.Mode` (`ModeAudit` default, `ModeEnforce`) with `Apply(closure.Decision) closure.Decision` that, in audit, downgrades a scope-escape `Block → Warn` (keeping `Escaping`/`Reason` for the report). `scope` imports `closure`, so this stays stdlib-only; `closure.Safe` is untouched. The CLI gains `--mode` (default `audit`) and maps the verdict to its existing exit-code scheme.

Alternatives weighed: (a) **pre-expand the ownership tree into resource clauses in the synthesizer** — rejected: the synthesizer would need `State` (loses purity/embeddability) and DESIGN §6 commits to an ownership *dimension*, not a flattened set. (b) **pass full `State` to `matchScope`** — rejected for the same reason ADR-0008 rejected it for selector (couples scope matching to the whole interface, hard to fake in tests); a narrow resolver is the minimal surface. (c) **apply mode inside `Safe`** — rejected: mode is a deployment concern, and the ADR fixes `Safe` as unchanged; keep it a caller-side transform.

## Steps (vertical slices)
1. [x] **`namespace` scope dimension** — `DimNamespace` + `NamespaceClause` + `Validate` + `matchScope` branch (pure `Ref`, gate-only); `closure` unit table; one golden using a hand-written `dim: namespace` scope. Built as an *explicit* dimension (DESIGN §6; v0.6 templates), **not** part of the Level-0 default. *Demo: an explicit "everything in prod" scope allows an intra-prod cascade, blocks a cross-namespace one.*
2. [x] **`ownership` scope dimension + resolver seam** — introduce the narrow resolver, port `matchScope` to it (resource/selector/namespace unchanged in behaviour), add `DimOwnership` + `OwnershipClause` + `Validate` + the owner→child membership walk (reusing `OwnedChildren`); `closure` unit table (root, child, grandchild in; sibling/unrelated out; cyclic-owner termination); one golden using a hand-written `dim: ownership` scope. *Demo: scope "the web Deployment and all it owns" allows the cascade in `01`.*
3. [x] **Level-0 synthesizer + loader/CLI wiring** — `scope.Derive`; `loadScope` derives when no scope file is present; CLI renders + reports provenance; `scope` unit test; a **new golden scenario** (no `scope.yaml`/`taskcontract.yaml`) proving the zero-config derived scope classifies an intra-namespace cascade correctly. *Demo: `krsm check <dir>` with no scope file at all returns a verdict.*
4. [x] **Audit/enforce mode** — `scope.Mode` + `Apply`; CLI `--mode` (default `audit`); a CLI/golden showing the same escape → `Warn` (audit) vs `Block` (enforce), with exit codes. *Demo: `krsm check --mode audit` flags, `--mode enforce` blocks.*

Each slice goes through `design-check-tdd` → `tdd`; `make check` green after each.

## Risks & unknowns
- **Resolver seam shape (highest).** The exact interface (one method per state-need vs a bundled resolver) is the design-doc's main job; getting it wrong now means another internal churn when `reference` lands. Mitigation: design for the *two* needs that exist (labels, ownership-subtree) with room for a third, and keep it unexported so it can change freely.
- **Ownership subtree semantics.** Must match `Closure`'s cascade exactly (uid-keyed owner match, depth-unbounded, cycle-safe). Mitigation: reuse `OwnedChildren` and the same visited-set pattern; a property test asserting `subtree(root) ⊆ R` and termination on cyclic owners.
- **Audit/provenance coupling.** ADR-0011 says audit softens *derived*-scope escapes; whether audit also softens *contract* escapes is a v0.5 webhook policy question. Mitigation: in v0.4 treat `--mode` as a global CLI switch and record the open question; do not bake provenance-conditional softening into the engine.
- **Cluster-scoped / cross-namespace false positives** in a derived namespace guard (shared-config namespaces). Mitigation: out of scope here (the allowlist is v0.5); the audit default makes any over-block visible, not disruptive.

## Definition of done
- `krsm check <dir>` with **no** `scope.yaml` and **no** `taskcontract.yaml` derives a Level-0 scope from the request target and prints `ACTION/SCOPE(derived)/CLOSURE/VERDICT`.
- The derived default **catches the intra-namespace cascade** (`01`-style) — proving it is the ownership tree, not a bare namespace boundary.
- `--mode audit` (default) renders a derived-scope escape as `Warn`; `--mode enforce` renders it as `Block`, with the existing exit codes.
- `closure.Safe` signature and the closure walk are unchanged; `internal/archguard` confirms `closure` and `scope` stay stdlib-only.
- `make check` green (`gofmt`, `go vet`, `golangci-lint`, `staticcheck`, `go test -race ./...`); all existing 22 goldens unchanged.

## Open questions
1. Resolver interface: a single `scopeResolver` with `labels(Ref)` + `ownedSubtree(root Ref) map[string]bool` (pre-expanded once per ownership clause), or lazy per-candidate `inSubtree(root, r Ref) bool`? (Design-doc to decide; pre-expansion is `O(tree)` once and likely simpler.)
- 2. Does `scope.Derive` belong in `scope` (compiled-predicate home, embeddable) or a new package? (Lean: `scope`, beside `Compile`.)
3. Mode placement: a `scope.Mode.Apply(Decision)` helper vs the CLI doing the downgrade inline. (Lean: `scope.Mode` so the webhook reuses it in v0.5.)
4. SCOPE-line rendering for ownership/namespace clauses — exact human form (`owns:…`, `ns:…/*`).
