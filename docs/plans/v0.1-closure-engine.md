# Plan — v0.1 Faithful Closure Engine

Status: **proposed** · Branch: `feat/closure-engine-v0.1` · Milestone: [ROADMAP §v0.1](../ROADMAP.md) · Design: [DESIGN.md §3–4, §7](../DESIGN.md)

This is the first vertical slice of KRSM: the `closure/` package that computes an
action's **affected-resource closure** `C(S,A)` over the four relations and decides
`Safe(T,S,A) ≡ C(S,A) ⊆ scope(T)`. It is the public, embeddable heart of the tool;
everything downstream (corpus golden tests, `TaskContract`, live reads, the webhook)
builds on it.

## Goal / done-when

The engine reproduces the **expected verdict and escaping set for every corpus
scenario** in `testdata/`, under `go test -race`. Scenario 1 (memory-pressure
cascade) is the tracer-bullet acceptance case and already exists as a fixture.

## Source of truth

- **Spec:** the Python reference oracle `~/krsm-scope-gate/scope_gate/closure.py`
  (BFS over effect-propagating relations) — port its *semantics*, not its types.
- **Behaviour corpus:** `~/krsm-scope-gate/corpus/failure-modes-v0.md` — the 10
  scope-escape scenarios + the coverage map (which relation each exercises).
- **Materialised fixture:** `~/krsm-scope-gate/fixtures/01-memory-pressure-cascade/`
  (`cluster.yaml`, `action.yaml`, `scope.yaml`, `expected.md`) — the exact golden
  format to generalise to `testdata/`.
- **Architecture:** DESIGN.md §3 (model `R,A,S,G(S),C,Safe,B`), §4 (closure algorithm
  + termination/decidability), §7 (repo layout: `closure/` is public/embeddable).

## Defects in the oracle that the Go port MUST fix

The prototype is correct enough to block fixture-01 but is **not faithful**. The Go
engine fixes these (they are the substance of "faithful"):

1. **Identity.** `Ref` is keyed on `kind` only → must be **GVK (group/version/kind) +
   namespace + name**, and resources are **matched by `uid`** where Kubernetes itself
   does (ownerReferences carry a `uid`).
2. **Owner matching** is by `(kind, name)` → must be by **`uid`** (corpus #1, #10:
   same-name objects in different tenants must not collide).
3. **Missing relations** (present in the corpus, absent in the engine):
   - **namespace containment** — deleting a Namespace affects all it contains (#5, #10).
   - **`envFrom` / `env[].valueFrom`** cross-refs to ConfigMap/Secret (#4, #7), not just
     volumes.
   - **finalizer → external** — removing `metadata.finalizers` flags the guarded
     external resource as a cross-boundary **WARN** (#9).
   - **`scaleTargetRef` / HPA** and **PDB** controller cross-refs on a scale action (#6).
   - **selector mutation** — a label/selector change unions the **old ∪ new** match-set
     (#2, #8); this needs the request payload (old vs new object).
4. **Effect-class gating per verb** — closure follows only the relations the action's
   effect class licenses (cascading-delete vs disruptive vs mutate-selector vs
   mutate-config vs finalizer-removal vs scale), per DESIGN §4.
5. **Verdict vocabulary** — distinguish **BLOCK** (closure escapes scope) from **WARN**
   (closure crosses the cluster boundary, e.g. finalizer→external, where in-cluster
   state can't confirm the effect).

## Package shape (proposed — design stage refines)

`closure/` (public SDK, no client-go dependency — operates on parsed/in-memory state so
it stays unit-testable and embeddable):

- model types: `GVK`, `Ref` (GVK+namespace+name, `UID`), `Object` (the per-relation
  fields), `Action` (verb, target, propagation, old/new payload), `EffectClass`.
- `State` / `G(S)`: indexed accessors — owned-children (by uid), pods-selected-by,
  selectors-targeting-pod, mounters/consumers (volume + envFrom + env), namespace
  contents, controller refs (scaleTargetRef/HPA, PDB). (v0.1 may scan; the *indexed*
  `O(c·d)` form is a later perf slice — DESIGN §4 "Performance". Keep the interface so
  the index can drop in without changing callers.)
- `Closure(s State, a Action) []Ref` — BFS worklist with visited-set guard; preserves
  `|C| ≤ |R|` and termination on cyclic/adversarial input (DESIGN §4, §8 property test).
- `Safe(scope []ScopeRef, c []Ref) (ok bool, escaping []Ref)` — finite subset test;
  `scope` supports the flat identity dimension (glob name) for now (richer
  `TaskContract` dimensions are v0.3).

## Test corpus → `testdata/` (golden files)

- Port the fixture-01 layout into per-scenario golden dirs under `closure/testdata/`
  (or `testdata/scenarios/`), each = `(cluster.yaml, action.yaml, scope.yaml,
  expected.yaml)` where `expected` carries `verdict`, `closure` set, and `escaping` set.
- A **table-driven** test loads each scenario dir and asserts closure + escaping +
  verdict. Adding a scenario = adding a directory (no test code change).
- v0.1 lands the relations the corpus exercises; materialise scenarios as the matching
  relation is implemented (TDD order below). Scenario 1 first.

## Build order (TDD vertical slices — red → green → refactor)

1. **Model + identity** (`Ref` with GVK+uid, `Object`, `Action`) + the table-driven
   scenario loader. Tracer bullet: **scenario 1** (ownerReference cascade + the
   `web-svc` selector binding) passes end-to-end → reproduces `expected.md` exactly.
2. **namespace containment** (#5, #10) + uid-based owner matching guards cross-tenant.
3. **label-selector** binding both directions + **selector mutation** old∪new (#2, #8).
4. **cross-resource refs**: volume + `envFrom` + `env.valueFrom` (#3, #4, #7), reverse
   (consumer-of-mutated-config).
5. **controller refs** scaleTargetRef/HPA + PDB on scale (#6); **finalizer→external**
   WARN (#9).
6. **property test**: `|C| ≤ |R|`, termination on a cyclic ownerRef fixture.

Each slice ends green under `go test -race` before the next starts.

## Out of scope for v0.1 (deferred, do not build here)

- `TaskContract` CRD → `ScopePredicate` compiler (v0.3) — v0.1 scope is a flat ref list.
- client-go informers / live reads (v0.4); the webhook (v0.5).
- The indexed `O(c·d)` performance engine and the 10²–10⁵ scale study (research
  apparatus, [ROADMAP §Deferred]). v0.1 may scan; keep accessors behind an interface.

## Constraints (binding)

- Idiomatic Go; **gofmt, golangci-lint, staticcheck, `go test -race ./...` all green**.
- Module path `github.com/ridik-il/krsm`; `closure/` and `scope/` are public/embeddable
  (no leaking client-go or internal types through their APIs).
- Where Go and the formal definitions could drift, **the formal definitions win**
  (DESIGN §11) — the code is reconciled to them.
- **Git: Claude does not commit or push.** Stage at most; the user commits under his own
  account (enforced by hook). Leave a one-line "ready to commit" summary.
