# Design: v0.3-taskcontract-compiler
Status: approved · Date: 2026-06-21 · Slug: `v0.3-taskcontract-compiler` · Plan: docs/plans/v0.3-taskcontract-compiler.md

## Summary
Introduce the user-facing scope contract and its compiler. A new **public, stdlib-only `scope/`
package** defines a Go `TaskContract` struct (DESIGN §6 shape) and `Compile(TaskContract)
(ScopePredicate, error)` that maps each declared allow-clause to a `closure.ScopeClause` (via the
slice-1 constructors), validates it, and fails closed on an unknown dimension or an unrecognised
`apiVersion`/`kind`. This slice supports the two dimensions that exist today — `resource` and
`selector` — and leaves room for the rest. YAML→`TaskContract` parsing lives in the
`internal/scenario` loader (which already depends on `sigs.k8s.io/yaml`), so `scope/` stays
YAML-free and embeddable. A new golden scenario driven by a real `TaskContract` YAML proves the
end-to-end path; `closure.Safe`'s signature is untouched.

## Interfaces / contracts

### `scope` package (new, public, embeddable, stdlib-only)
```go
package scope

// TaskContract is the declarative, agent-referenced authorised scope (ADR-0003,
// DESIGN §6) as a Go value. Parsing its YAML wire form is a loader concern; this
// package compiles the struct, so the embeddable SDK pulls in no YAML dependency.
type TaskContract struct {
    APIVersion string   // must be "krsm.io/v1"
    Kind       string   // must be "TaskContract"
    Metadata   Metadata
    Spec       Spec
}
type Metadata struct{ Name, Namespace string }
type Spec struct {
    Allow       []AllowClause
    MaxSeverity Severity
}

// AllowClause is one declared scope dimension (the wire shape). Exactly one
// dimension's fields are meaningful, chosen by Dim — resource and selector now;
// ownership/namespace/reference fields are added by later slices.
type AllowClause struct {
    Dim       closure.ScopeDim   // "resource" | "selector"
    GVK       closure.GVK
    Namespace string
    Name      string             // Dim == resource
    Selector  closure.LabelSelector // Dim == selector (matchLabels + matchExpressions)
}

// Severity mirrors DESIGN §6 maxSeverity. Carried on the predicate; NOT enforced
// in this slice (severity B is a later concern).
type Severity string
const (
    SeverityNone     Severity = ""
    SeverityLow      Severity = "low"
    SeverityMedium   Severity = "medium"
    SeverityHigh     Severity = "high"
    SeverityCritical Severity = "critical"
)

// ScopePredicate is a compiled TaskContract: the clauses closure.Safe consumes,
// plus the (unenforced) maxSeverity. Callers pass predicate.Clauses to Safe.
type ScopePredicate struct {
    Clauses     []closure.ScopeClause
    MaxSeverity Severity
}

// Compile lowers a TaskContract to a ScopePredicate, or fails closed. It errors on
// an unrecognised apiVersion/kind, an unknown dimension, an invalid clause
// (closure.ScopeClause.Validate), or an unknown maxSeverity — an uncompilable
// contract must never silently yield an empty (allow-nothing) or partial scope.
func Compile(tc TaskContract) (ScopePredicate, error)
```
- **`Compile` errors** (all wrapped, fail-closed): `apiVersion != "krsm.io/v1"`; `kind !=
  "TaskContract"`; an `AllowClause.Dim` not in {`resource`,`selector`} (incl. the not-yet-built
  `ownership`/`namespace`/`reference` → explicit "unsupported dimension" error, not silent skip);
  a clause that fails `closure.ScopeClause.Validate()`; a `MaxSeverity` outside the known set.
- **Dim→clause mapping** is the one place that knows dimensions: `resource` →
  `closure.ResourceClause(gvk, ns, name)`; `selector` → `closure.SelectorClause(gvk, ns, selector)`.
  Adding a dimension later is a new `case` here + new `AllowClause` fields — no caller changes.

### `closure` package — unchanged
`Safe(s State, a Action, scope []closure.ScopeClause) Decision` is **not** touched. The compiled
`ScopePredicate.Clauses` is what callers hand to it.

## Data model

### `taskcontract.yaml` wire form (DESIGN §6) — new corpus artifact
```yaml
apiVersion: krsm.io/v1
kind: TaskContract
metadata: { name: restart-frontend-web-pods, namespace: prod }
spec:
  allow:
    - dim: resource
      gvk: { group: apps, version: v1, kind: Deployment }
      namespace: prod
      name: frontend
    - dim: selector
      gvk: { version: v1, kind: Pod }
      namespace: prod
      matchExpressions:
        - { key: app, operator: In, values: [web] }
  maxSeverity: high
```
- The loader's `rawTaskContract` mirrors this (apiVersion, kind, metadata, spec.allow[], maxSeverity);
  each raw allow-clause reuses the existing `matchLabels` wrapper parse for the selector dim.

## Behavior
- **Happy path:** a TaskContract with a resource clause + a selector clause compiles to two
  `ScopeClause`s; `closure.Safe(state, action, predicate.Clauses)` returns the same verdict the
  hand-written `scope.yaml` did.
- **Unknown/forward dimension:** `dim: ownership` (valid future syntax, unbuilt) → `Compile` errors
  "unsupported scope dimension". This is stricter than the engine's defensive "match nothing": at the
  *contract* boundary an unbuildable dimension is a hard error so an author learns immediately, rather
  than getting a silently narrowed scope.
- **Bad envelope:** `apiVersion: v1` or `kind: Pod` → error. Empty `spec.allow` compiles to an empty
  predicate (a contract that authorises nothing) — legal and fail-safe (everything escapes → Block),
  not an error.
- **Loader preference:** scenario with `taskcontract.yaml` → compiler path; scenario with only
  `scope.yaml` → legacy path. The 19 legacy scenarios are unaffected.

## Components & responsibilities
- **`scope/scope.go`** (new) — `TaskContract`/`Spec`/`AllowClause`/`Severity`/`ScopePredicate` types
  and `Compile`. Imports `closure` only (stdlib otherwise). Owns the dim→clause mapping and all
  fail-closed contract validation.
- **`scope/scope_test.go`** (new, `package scope`) — Compile behaviour via the public API.
- **`internal/scenario/scenario.go`** — `parseTaskContract` + `rawTaskContract`; `Load` prefers
  `taskcontract.yaml`. The only place importing both `scope` and `sigs.k8s.io/yaml`.
- **`internal/archguard/archguard_test.go`** — add `TestScopeIsStdlibOnly`, a direct parallel of
  `TestClosureIsStdlibOnly` over `github.com/ridik-il/krsm/scope/...`. The existing check already
  `continue`s on the `github.com/ridik-il/krsm` module prefix, so the intra-module `scope→closure`
  import passes while an external (`k8s.io/...`) leak fails — no change to the matching logic, just a
  second target package.
- **`closure/testdata/scenarios/21-taskcontract-selector-scope/`** (new) — `cluster.yaml` /
  `request.yaml` (mirroring scenario 20's selector-precision setup) + `taskcontract.yaml` (above) +
  `expected.yaml` (Allow, same three-ref closure). Proves the compiler path end-to-end through the
  directory-driven golden test. *(Refines plan Q2: a **new** scenario rather than converting 20, so
  scenario 20 keeps golden coverage of the `scope.yaml` selector path.)*
- **`CHANGELOG.md`** — `scope` package + TaskContract compiler. **`docs/adr/0009-...`** — the decision.

## Alternatives considered
- **`Compile` takes YAML bytes** — rejected: pulls a YAML dependency into the embeddable `scope`
  SDK, the same boundary violation ADR-0005/§7 guard for `closure`. Parsing stays in the loader.
- **`ScopePredicate = []closure.ScopeClause` (alias)** — rejected: no home for `maxSeverity` or
  future per-predicate metadata; a named struct costs nothing now and avoids a later reshape.
- **Reuse `closure.ScopeClause` directly as the wire/contract type** — rejected: `ScopeClause` is the
  *compiled* engine input; the `TaskContract`/`AllowClause` is the *authored* contract (envelope,
  metadata, maxSeverity, room for un-compiled dims). Conflating them couples the wire schema to the
  engine type and loses the compile/validate seam.
- **Change `closure.Safe` to take a `ScopePredicate`** — rejected: a second back-to-back public break
  for no gain; `Safe` stays clause-based and `scope` sits above it.
- **Convert scenario 20 to a TaskContract** (plan's initial lean) — rejected: would drop golden
  coverage of the `scope.yaml` selector path; a new scenario 21 adds the compiler proof additively.
- **Silently skip an unknown dimension in `Compile`** — rejected: a security control must fail closed
  on an authored-but-unbuildable scope; a hard error tells the author, an empty scope would mislead.

## Trade-offs & risks
- **Optimises for** a clean embeddable contract boundary (DESIGN §7), reuse of the slice-1 engine
  (no second scope model), and an end-to-end golden proof; **costs** a new package + loader branch +
  a near-duplicate scenario.
- **Risk — golden regression.** Mitigation: the loader change is additive (legacy `scope.yaml` path
  untouched); all 20 existing scenarios stay byte-for-byte; scenario 21 is new.
- **Risk — `scope/` accidentally pulls a non-stdlib dep.** Mitigation: `TestScopeIsStdlibOnly`.
- **Risk — TaskContract YAML drifts from DESIGN §6.** Mitigation: the corpus artifact is the literal
  §6 shape (`krsm.io/v1`, `kind: TaskContract`, `spec.allow`, `maxSeverity`); compile rejects a wrong
  envelope.
- **Risk — `maxSeverity` parsed but unused invites "dead field" confusion.** Mitigation: documented
  as carried-not-enforced (severity B is a later slice); validated against the known set so a typo is
  caught now.

## Testability
Observed through the public `scope.Compile`, the `internal/scenario` loader, and the golden corpus —
never through unexported helpers.
- **`scope.Compile` (package scope):**
  - a TaskContract with a resource + a selector clause → `ScopePredicate.Clauses` equals the expected
    `closure.ScopeClause`s (built via the constructors), `MaxSeverity` carried;
  - `dim: ownership` (or any unknown dim) → error;
  - `apiVersion`/`kind` wrong → error; unknown `maxSeverity` → error;
  - empty `spec.allow` → empty predicate, no error;
  - a clause that fails `ScopeClause.Validate` (e.g. selector clause with a `Name`) → error.
- **Loader + golden (scenario 21):** `Load` over the `taskcontract.yaml` scenario yields a `Scenario`
  whose `Safe` verdict is `Allow` with the expected closure — the compiler path classifies correctly.
  A scenario with only `scope.yaml` still loads via the legacy path (the 19 goldens).
- **`archguard`:** `TestScopeIsStdlibOnly` green (scope imports only stdlib + own module).
- **Regression:** all 20 existing goldens green; `closure.Safe` signature unchanged; full `make
  check` green.

## Open questions
None blocking — plan's five resolved (ScopePredicate struct; optional `taskcontract.yaml` with
`scope.yaml` fallback; `Safe` unchanged; `maxSeverity` carried-not-enforced; wrong envelope →
error, recorded as **ADR-0009**). One refinement logged above: a **new scenario 21** rather than
converting scenario 20. Deferred to later v0.3 slices (explicit non-goals): the `ownership`,
`namespace`, `reference` dimensions and inexpressible-scope gap signalling.
