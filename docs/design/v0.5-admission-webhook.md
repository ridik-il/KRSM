# Design: v0.5-admission-webhook
Status: draft · Date: 2026-06-24 · Slug: `v0.5-admission-webhook` · Plan: docs/plans/v0.5-admission-webhook.md

## Summary
v0.5 turns KRSM from an offline/one-shot CLI checker into the **product**: a
`ValidatingWebhook` that computes the affected-resource closure of an agent's proposed
action against **live, informer-backed, indexed** cluster state and flags (audit) or
denies (enforce) the action *before it persists* (DESIGN §5, ADR-0004, ADR-0011). It does
this by adding two non-embeddable packages outside the stdlib-only `closure`/`scope`
trees:

1. **`state/`** — an informer-fed, incrementally **indexed** `closure.State`
   implementation (the second `State` beside `closure.NewScanState` and
   `internal/cluster`), maintaining four inverted indexes (owner, namespace, selector,
   reverse cross-ref) so closure costs `O(c·d)` in blast radius, not `O(c·n)` in cluster
   size (DESIGN §4). It reuses the **same per-object relation projection** as the v0.4
   one-shot reader (factored out in slice 2), so goldens 01–27 stay the authoritative
   correctness spec via a parity oracle.
2. **`webhook/`** — an HTTPS `AdmissionReview` handler that extracts the `closure.Action`
   from the request, resolves `scope(T)` through the progressive channel (derived /
   annotation / contract), runs the **unchanged** `closure.Safe` → `scope.Mode.Apply`, and
   answers allow/deny with a reason + provenance. **Fail-closed** on every uncertainty.

The same milestone **resolves the v0.4 real-world read gaps** found in the PR #11 review:
the informer path is the consistent, bounded, indexed read that fixes C2/C3/S3, the
`AdmissionReview` payload moots S2/S4, and explicit handling fixes C1/S1. `closure.Safe`,
the closure walk, the derived-scope/mode code, and the golden corpus are **untouched** —
v0.5 is new state + transport around the existing decision procedure.

## The v0.4 real-world gaps and how v0.5 fixes each (must-fix carry-over)
The PR #11 critical review found seven real-world failure modes in the v0.4 one-shot
reader. v0.5 resolves all of them; this table is the contract for that.

| ID | v0.4 gap (file) | v0.5 fix | Slice |
|---|---|---|---|
| **C1** | Partial discovery (`ErrGroupDiscoveryFailed`) is fatal → deny-all (`internal/cluster/reader.go:99`) | Tolerate partial discovery: keep resolved groups; fail-closed **only** if a *closure-relevant* group failed. Applied to both the one-shot reader and the informer factory startup. | 1, 3 |
| **C2** | Non-atomic multi-List → torn snapshot → false ALLOW (`reader.go:111`) | Single informer-fed cache (one consistent watch stream per resource) + a per-request **staleness guard**: compare the request object's `resourceVersion` to cache, bounded `GET` over the `O(d)` closure neighbourhood on drift, **fail-closed** on unresolved drift (ADR-0004). | 3, 4 |
| **C3** | Unbounded whole-cluster List incl. every Secret `data` (`reader.go:113`) | Informer list-watch paginates internally; **metadata-only** (`PartialObjectMetadata`) informer for Secrets/ConfigMaps so `data` is never cached; `Limit` pagination added to the one-shot path too. | 1, 3 |
| **S1** | No timeout/cancellation (`cmd/krsm/main.go:260`) | CLI: `--timeout` + `signal.NotifyContext`. Webhook: per-request deadline (≤ the API server's `timeoutSeconds`) + fail-closed on timeout. | 1, 5 |
| **S2** | Group-blind target resolution collides across CRD groups | **Moot on the webhook path** — the `AdmissionRequest` carries the exact `resource`/`kind` GVK and the object's uid; no discovery-guess resolution. Documented as a known CLI limitation (add `Kind.group` qualifier later). | 5 |
| **S3** | Multi-version resources double-listed (`selectTargets` dedups only by GVR) | List/inform **preferred versions only** (`ServerPreferredResources` / one informer per preferred GVR). | 1, 3 |
| **S4** | Cascade hardcoded `true`; real `propagationPolicy` not expressible | The `AdmissionReview` for a DELETE carries `options` (`metav1.DeleteOptions.propagationPolicy`); the handler maps Orphan→`Cascade:false`, Background/Foreground→`Cascade:true`. | 5 |

## Interfaces / contracts

### `state/` — the informer-backed indexed `closure.State` (slice 3)
```go
package state

import (
    "context"
    "k8s.io/client-go/rest"
    "github.com/ridik-il/krsm/closure"
)

// Provider is an informer-backed, incrementally indexed closure.State (ADR-0004,
// DESIGN §4). It is the SECOND closure.State implementation (beside closure.NewScanState
// and the v0.4 one-shot internal/cluster reader), satisfying the SAME interface so the
// engine, scope, and goldens are unchanged. Closure costs O(c·d) (blast radius), not
// O(c·n) (cluster size), because each relation is served from an inverted index.
//
// Read-only: it runs List+Watch (get/list/watch) over the tracked GVRs and NEVER issues
// a write verb. Secrets/ConfigMaps are watched METADATA-ONLY (PartialObjectMetadata) so
// their data is never fetched or cached.
type Provider struct { /* informer factories, the four indexes, a rest client for fallback */ }

// New builds a Provider from a *rest.Config and options (tracked GVRs come from the
// hardened discovery selection; mode/allowlist are webhook concerns, not state's). It
// does NOT block; call Start then WaitForCacheSync before serving.
func New(cfg *rest.Config, opts Options) (*Provider, error)

// Start launches the informers. WaitForCacheSync blocks until every tracked informer's
// cache is initially populated; the webhook MUST NOT serve verdicts before it returns
// true — an unsynced cache is an unknown closure (fail-closed, DESIGN §5).
func (p *Provider) Start(ctx context.Context)
func (p *Provider) WaitForCacheSync(ctx context.Context) bool

// Provider implements every closure.State method from its indexes:
//   Get, OwnedChildren, NamespaceContents, PodsSelectedBy, PodsMatching,
//   SelectorsTargeting, SelectorsMatchingLabels, Consumers, ControllersTargeting.
var _ closure.State = (*Provider)(nil)

// FreshGet does a bounded, on-demand live GET for one Ref, used by the staleness guard
// (slice 4) to confirm a closure-neighbourhood member when the cache looks stale. It is
// the ADR-0004 "GET fallback", scoped to O(d) members — never a re-list.
func (p *Provider) FreshGet(ctx context.Context, ref closure.Ref) (closure.Object, bool, error)
```

#### The four indexes (maintained by informer add/update/delete handlers)
| Index | Shape | Serves | Built from |
|---|---|---|---|
| owner | `uid → []childRef` | `OwnedChildren` | `metadata.ownerReferences[].uid` |
| namespace | `ns → []ref` | `NamespaceContents` | `metadata.namespace` + discovered scope |
| selector (inverted) | `ns → []selectorOwnerRef` + per-owner `LabelSelector` | `PodsSelectedBy`, `SelectorsTargeting`, `SelectorsMatchingLabels`, `PodsMatching` | `spec.selector`/`podSelector` + Pod `labels` |
| reverse cross-ref | `referentKey → []consumerRef` | `Consumers`, `ControllersTargeting` | volumes/env/imagePullSecrets/`scaleTargetRef`/finalizers |

Each handler updates the indexes under a single `sync.RWMutex`; reads (the nine `State`
methods) take the read lock and return slices of `closure.Ref`. The indexes store only
`closure.Object`-shaped projections (the per-relation fields), never raw API objects, so
Secret/ConfigMap `data` is structurally absent from the cache.

### Shared per-object projection (slice 2 — refactor, no behaviour change)
```go
// project extracts the four relations from one unstructured object into a closure.Object,
// using the SAME field paths as internal/cluster.BuildObjects. Factored out so the
// one-shot reader (v0.4) and the informer index (v0.5) produce IDENTICAL closure.Objects
// from identical input — the parity guarantee that keeps goldens authoritative. Lives in
// internal/cluster (or a shared internal/relations) and is consumed by both.
func project(obj *unstructured.Unstructured, scope ScopeInfo) (closure.Object, error)
```

### `webhook/` — the admission handler (slices 5–6)
```go
package webhook

import (
    "net/http"
    admissionv1 "k8s.io/api/admission/v1"
    "github.com/ridik-il/krsm/closure"
    "github.com/ridik-il/krsm/scope"
)

// Server handles ValidatingWebhook AdmissionReviews. It holds the indexed state, the
// scope resolver, the mode, and the cross-boundary allowlist. It NEVER mutates the
// request (validating, not mutating — DESIGN §1/§5).
type Server struct { /* state closure.State (the Provider), mode scope.Mode, resolver scopeResolver, allowlist, taskContractLister, agentMatcher */ }

// Handle is the core: decode+validate the AdmissionReview, extract the Action, resolve
// scope, run the SHARED closure.Safe → scope.Mode.Apply, and build the AdmissionResponse.
// It is pure over its inputs (testable with crafted AdmissionReview JSON + a fake state)
// and FAIL-CLOSED: any decode/validation/internal error or a not-synced cache yields
// allowed=false with a credential-free reason (enforce) / a warning (audit per the mode).
func (s *Server) Handle(review admissionv1.AdmissionReview) admissionv1.AdmissionReview

// ServeHTTP wires Handle to the TLS endpoint with a per-request deadline (≤ the webhook
// config's timeoutSeconds); a handler panic recovers to a fail-closed deny.
func (s *Server) ServeHTTP(w http.ResponseWriter, r *http.Request)

// actionFromRequest builds the closure.Action from an AdmissionRequest: Verb from
// request.Operation (DELETE/UPDATE/...), Target from request.Resource+Name+Namespace+UID
// (REAL GVK+uid — no discovery guess, fixing S2), Cascade from DeleteOptions
// propagationPolicy (Orphan→false, fixing S4), and Old/New from oldObject/object (enabling
// the selector/finalizer/config-mutation verdicts the CLI could not express).
func actionFromRequest(req *admissionv1.AdmissionRequest) (closure.Action, error)
```

### Scope resolution (slice 6 — reuses `scope`, no new dimensions)
```go
// resolveScope picks the scope provenance for a request (ADR-0011) and returns the
// clauses + a provenance string for the response:
//   Level 3 contract  — a TaskContract referenced by annotation/label, resolved from an
//                        informer → scope.Compile.                 provenance "contract"
//   Level 1 annotation— krsm.io/target: "<Kind>/<ns>/<name>" → scope.Derive(thatTarget).
//                                                               provenance "annotation"
//   Level 0 derived   — no contract/annotation → scope.Derive(request target).
//                                                               provenance "derived:ownership-tree"
// Level 2 templates are v0.6 (out of scope). In derived (Level 0/1) modes only, the
// cross-boundary allowlist removes allowlisted namespaces / cluster-scoped kinds from the
// escaping set before the verdict (ADR-0011 escape hatch); a contract ignores it.
func (s *Server) resolveScope(req *admissionv1.AdmissionRequest, target closure.Ref) ([]closure.ScopeClause, string, error)
```

## Data model

### From `AdmissionRequest` to `closure.Action` (the fidelity the CLI lacked)
| Action field | AdmissionRequest source | Note |
|---|---|---|
| `Verb` | `request.operation` (`DELETE`/`UPDATE`/`CREATE`/`CONNECT`) → `closure.Verb` | CREATE has no closure (nothing exists yet) → admit fast; KRSM gates mutation/deletion |
| `Target.GVK` | `request.resource` / `request.kind` | **real** group/version/kind — fixes S2 |
| `Target.{Namespace,Name,UID}` | `request.namespace`/`name`; `request.oldObject.metadata.uid` | real uid → exact `State.Get` |
| `Cascade` | `request.options` (`DeleteOptions.propagationPolicy`) | Orphan→false, Background/Foreground→true — fixes S4 |
| `Old` | `request.oldObject` (projected via `project`) | enables selector/finalizer/config-mutation old∪new match-set |
| `New` | `request.object` (projected via `project`) | nil for DELETE |

### `AdmissionResponse`
- `allowed` = `false` iff the applied `Decision.Verdict == closure.Block` (enforce, or a
  fail-closed deny in any mode); `true` otherwise (allow, or an audit-downgraded warn).
- `status.message` / `warnings[]` carry the human reason: the verdict, the **scope
  provenance**, and the escaping refs (DESIGN §5: distinguish *"closure escapes"* from
  *"cannot compute closure"*). Never any credential material.
- `uid` echoes `request.uid` (required); `patch` is never set (validating, not mutating).

### Tracked GVRs and the informer split
- **Full informers** (need labels/spec fields): Pods, workloads (Deployment/ReplicaSet/
  StatefulSet/DaemonSet/Job/CronJob), Services, PDBs, NetworkPolicies, HPAs, Namespaces,
  PVCs, plus discovered CRDs that participate via ownerReferences.
- **Metadata-only informers** (`PartialObjectMetadata`, no `data`): Secrets, ConfigMaps —
  they appear in the closure only as *referenced* objects (cross-ref relation), and
  `PartialObjectMetadata` still carries `labels`/`ownerReferences`/`finalizers`, which is
  all the relations read. Their `data` is never fetched (fixes C3).

## Behavior

### Request lifecycle (per `Handle`)
1. Decode the `AdmissionReview`; **strict schema validation** (reject malformed → fail
   closed). Reject if `request` is nil or `kind`/`resource` is empty.
2. **Scope the request:** if it does not match the agent-identity matcher (label/annotation
   + the namespaceSelector already filtered most at the API server), **admit** (KRSM gates
   only agent-originated requests — DESIGN §5).
3. CREATE → admit (no existing closure). Otherwise build `Action` (`actionFromRequest`).
4. **Cache-sync gate:** if `WaitForCacheSync` has not completed, **fail closed** (unknown
   closure).
5. Compute `Closure`/`Decision` = `closure.Safe(state, action, clauses)` where `clauses`
   come from `resolveScope`.
6. **Staleness guard (slice 4):** if the request's target `resourceVersion` is newer than
   the cached copy, `FreshGet` the closure neighbourhood; if it still cannot be reconciled,
   **fail closed** with a distinct reason.
7. Apply the mode: `dec = mode.Apply(dec)` (audit downgrades a scope-escape Block to Warn;
   a fail-closed Block is never downgraded — `scope.Mode.Apply` already guarantees this).
8. Build the `AdmissionResponse` (allowed + reason + provenance + escaping/warnings).

### Fail-closed taxonomy (DESIGN §5/§9 — each a distinct, credential-free reason)
- **Cache not synced** → deny "state not ready" (audit: a hard deny too — an unknown
  closure is not a derived-scope escape, so it is *not* softened, matching `Mode.Apply`).
- **Closure escapes scope** → enforce deny / audit warn (the expected, explained case).
- **Unresolvable target** (absent from state) → `Safe` empty-closure deny.
- **Staleness drift unresolved** → deny "could not confirm current state".
- **Decode/validation/internal error/timeout/panic** → deny "internal error".

### Audit-first default (ADR-0011)
Install default `mode=audit`: a *derived-scope* escape returns `allowed:true` with a
warning naming the escaping resources and the provenance — never a deny, so a day-0 false
positive does not get KRSM uninstalled. `mode=enforce` returns the Block as a real
`allowed:false`. A **fail-closed** deny (unknown closure) is a hard deny in *both* modes.

## Components & responsibilities
- `state/provider.go` — the `Provider`, informer factories (full + metadata), `Start`/
  `WaitForCacheSync`, the nine `closure.State` methods over the indexes, `FreshGet`.
  Read-only; no write verb (a source-guard test mirrors v0.4's `TestPackageInvokesNoWriteVerbs`).
- `state/index.go` — the four inverted indexes + their event handlers; the shared
  `project` consumer.
- `internal/cluster/build.go` (or `internal/relations/`) — `project` factored out of
  `BuildObjects` (slice 2), consumed by both reader and index.
- `internal/cluster/reader.go` — C1/S3/C3/S1 hardening (slice 1): partial-discovery
  tolerance, `ServerPreferredResources`, `Limit` pagination.
- `webhook/server.go` — `Server`, `Handle`, `ServeHTTP`, `actionFromRequest`, the
  fail-closed response builder, the per-request deadline + panic recovery.
- `webhook/scope.go` — `resolveScope` (Level 0/1/3), the TaskContract informer/lister, the
  cross-boundary allowlist.
- `webhook/admission_test.go` — `Handle` over crafted `AdmissionReview` JSON + a fake
  `closure.State`; the fail-closed taxonomy; the audit/enforce split.
- `cmd/krsm/` — a `serve` subcommand (`krsm serve --mode audit|enforce --tls-cert … --tls-key …`)
  that builds the `Provider`, waits for sync, and starts the `Server`. The existing
  `check` subcommand is unchanged (CLI dir + v0.4 live path).
- `deploy/` — `ValidatingWebhookConfiguration` (tight `namespaceSelector`/`objectSelector`,
  `failurePolicy: Fail`), the webhook `Deployment`/`Service`, cert (cert-manager
  `Certificate` + self-bootstrap CA fallback), the extended least-privilege RBAC
  (informer set get/list/watch + TaskContract), and a Helm chart (audit default).
- `deploy/crd/` — the `TaskContract` CRD (`krsm.io/v1alpha1`).
- `internal/e2e/webhook_e2e_test.go` — the `//go:build cluster` v0.5 acceptance test.
- `internal/archguard` — unchanged guard; confirm `state`/`webhook` deps don't leak into
  `closure`/`scope`.

## Security & deployment (DESIGN §9 — the webhook is itself an attack surface)
- **TLS only.** The server never serves plaintext; the `caBundle` in the webhook config
  matches the serving cert. cert-manager `Certificate` preferred (auto-rotation); a
  self-signed CA bootstrap Job patches the `caBundle` for clusters without cert-manager.
- **Least privilege, read-only.** RBAC grants only get/list/watch over the tracked GVRs +
  the TaskContract CRD — no write, no wildcard (extends `deploy/rbac/krsm-reader-clusterrole.yaml`).
  A source-guard test forbids any write verb in `state`/`webhook`.
- **Tight match + fail-closed, carefully.** `failurePolicy: Fail` (DESIGN §5) but the
  `namespaceSelector` **excludes** `kube-system` and KRSM's own namespace and the
  `objectSelector`/identity matcher restricts to agent-originated requests, so KRSM being
  down cannot wedge the control plane. A documented **break-glass** (delete the
  `ValidatingWebhookConfiguration`) and a readiness probe gated on cache-sync.
- **Strict input validation.** Every `AdmissionReview` is schema-validated; the handler
  holds itself to the standard it enforces.
- **No credential leakage.** Reasons/warnings/logs never echo the `rest.Config`, tokens,
  or object `data`; Secrets are metadata-only in cache.
- **Scope-channel trust (deferred, documented).** A compromised agent could reference an
  over-broad `TaskContract` (ADR-0003 open question). v0.5 *reports* provenance but does
  not yet verify issuer identity; tracked in SECURITY.md as the threat-model follow-up.

## Alternatives considered
- **Build the webhook on the v0.4 one-shot reader.** Rejected: a per-request whole-cluster
  list is `O(n)` per admission and re-triggers C2/C3 on every agent action — DESIGN §4 and
  ADR-0004 already chose informers. The one-shot reader stays the CLI's path only.
- **controller-runtime for the webhook + manager.** Rejected (plan decision 5): KRSM needs
  an HTTPS handler + informers + admission decode, all in client-go/api already; the
  manager/reconciler surface is unused weight and `govulncheck` area. Revisit if cert/CRD
  plumbing proves heavy.
- **Full objects in cache for everything.** Rejected (C3): Secrets/ConfigMaps as
  `PartialObjectMetadata` removes the secret-data exposure and shrinks memory, and the
  relations never read their `data`.
- **A global consistent snapshot per request.** Impossible — Kubernetes has no
  multi-resource snapshot primitive. The staleness guard (bounded neighbourhood GET +
  fail-closed on drift) is the pragmatic, honest bound (ADR-0004).
- **Mutating webhook that annotates/blocks.** Rejected — DESIGN §1/§5: KRSM decides, it
  does not rewrite; validating only.

## Trade-offs & risks
(See the plan's Risks section; the headline ones.)
- **Staleness window (ADR-0004).** Bounded, not eliminated; documented; low QPS makes the
  fallback affordable. The research-relevant underdetermination knob.
- **Index/loader parity.** The corpus parity oracle (slice 3) gates it before any cluster.
- **A fail-closed webhook can wedge the cluster.** Mitigated by the tight agent-only match
  excluding control-plane namespaces + a documented break-glass.
- **Cert rotation breaking admission.** cert-manager preferred; startup caBundle check.
- **Larger dep tree / new admission attack surface.** Resolve advisories by bump; strict
  validation; note in the PR.

## Testability
Hermetic-first (fake clients, crafted `AdmissionReview`s); one `kind` e2e anchors the
milestone. Test list (handed to `design-check-tdd` to order):
- **Hardening (slice 1):** partial discovery (`ErrGroupDiscoveryFailed`) still yields a
  verdict for an unrelated target; a closure-relevant failed group fails closed; a
  multi-version resource is listed once (`ServerPreferredResources`); a `List` is paginated;
  the CLI honours `--timeout`.
- **Projection parity (slice 2):** `project` and `BuildObjects` agree object-for-object on
  the corpus fixtures.
- **Indexed State (slice 3):** each index/`State` method over a fake informer; the indexed
  `Provider` reproduces goldens 01–27's closure/escaping set (parity oracle); a
  metadata-only Pod still binds a Service selector; Secret objects carry no `data` in cache;
  serving before `WaitForCacheSync` fails closed.
- **Staleness (slice 4):** a request newer than cache triggers a bounded `FreshGet`, not a
  re-list; unresolved drift fails closed with the distinct reason.
- **Webhook (slice 5):** `Handle` over a crafted escaping `delete` review → `allowed:false`
  (enforce) / `allowed:true`+warning (audit); CREATE admitted; a non-agent request admitted;
  Orphan `DeleteOptions` → `Cascade:false` (S4); an UPDATE with selector change → old∪new
  match-set; malformed review / not-synced / panic → fail-closed deny; `uid` echoed; no
  `patch`.
- **Scope channel (slice 6):** the same action Allowed under a contract authorising the
  tree, Warned under derived audit, Denied under derived enforce; `krsm.io/target`
  annotation re-roots the derivation; the cross-boundary allowlist exempts an allowlisted
  namespace in derived mode but not under a contract; provenance string correct in each.
- **Deployment (slice 7):** `helm template` renders a valid, audit-default config with a
  `kube-system`-excluding `namespaceSelector` and `failurePolicy: Fail`; RBAC has no
  write/wildcard verb (a manifest lint test).
- **e2e (slice 8, `//go:build cluster`):** on kind, agent-stamped escaping `delete` →
  flagged in audit / denied pre-persistence in enforce; in-scope admitted; both derived and
  full-contract; in-scope/escaping match goldens.

## Open questions
1. controller-runtime vs raw client-go (lean raw; revisit on cert/CRD plumbing weight).
2. cert-manager default vs self-bootstrap default for the quickstart.
3. Staleness fallback: always-GET-target vs drift-triggered (lean drift-triggered).
4. `TaskContract` CRD version: `krsm.io/v1alpha1` now, reserve `v1` (DESIGN §6 shows `v1`).
5. Single-replica (lean, audit-first) vs leader-elected from the start.
6. Cross-boundary allowlist location: webhook startup config (lean) vs the CRD.
7. CREATE handling: always-admit (lean — no existing closure) vs gate creates that adopt
   existing children via a pre-set ownerReference (a rare escape vector — defer, note it).
