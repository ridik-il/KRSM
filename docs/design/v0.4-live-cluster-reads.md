# Design: v0.4-live-cluster-reads
Status: draft · Date: 2026-06-23 · Slug: `v0.4-live-cluster-reads` · Plan: docs/plans/v0.4-live-cluster-reads.md

## Summary
Add a new **`internal/cluster`** package — outside the stdlib-only `closure`/`scope`
trees — that builds the four relations (DESIGN §3) from live cluster objects read as
`unstructured.Unstructured`, then plugs the result into the existing engine via the
unchanged `closure.State` seam. The package has two responsibilities, split so the v0.5
webhook can reuse the second half: (1) a **reader** (discovery + RESTMapper + dynamic
client) that lists the relevant GVKs and reports each GVK's namespaced/cluster-scoped
scope, and (2) a **pure index builder** — `BuildObjects([]unstructured.Unstructured,
ScopeInfo) ([]closure.Object, error)` — that ports the `internal/scenario` relation
extractors (`crossRefsFrom`, `selectorFrom`, `podSpecCrossRefs`, owner/finalizer reading)
to read unstructured maps via `NestedSlice`/`NestedMap`/`NestedString`, keeping the
**same field paths** as the YAML loader. Two things change versus the loader: uids are
**real** (`metadata.uid` for `Ref.UID`, `ownerReferences[].uid` for `OwnerRef.UID` — no
`uidOf` synthesis, no name-based owner resolution), and cross-ref uids are resolved
through a **name→uid index** built over the listed objects. `closure.NewScanState(objs)`
is reused verbatim, so `closure.Safe`'s signature, the closure walk, and goldens 01–26
are untouched. **This design doc covers the whole package; SLICES 1 and 2 (the pure
builder + name→uid resolver, and the discovery-backed `ScopeInfo` + read-only `Reader`)
are implemented now** — the CLI live path, full fail-closed deny reasons, and `kind` e2e
are slices 3–5 (deferred, sections marked *[slice N — deferred]*).

## Interfaces / contracts

### `internal/cluster` — the pure index builder (SLICE 1, this slice)
```go
package cluster

import (
    "k8s.io/apimachinery/pkg/apis/meta/v1/unstructured"
    "github.com/ridik-il/krsm/closure"
)

// ScopeInfo reports, per GVK, whether that kind is namespaced (true) or
// cluster-scoped (false). The live reader fills it from discovery/RESTMapper
// (slice 2); slice-1 tests build it by hand. It is the live replacement for the
// loader's static clusterScopedKinds map (scenario.go:249).
type ScopeInfo interface {
    // Namespaced reports whether gvk is a namespaced kind. ok is false when the
    // GVK's scope is unknown (not discovered): the caller must fail closed
    // (slice 2/4) rather than guess — guessing cluster-scoped would let a
    // namespaced object escape namespace containment (the unsafe direction).
    Namespaced(gvk closure.GVK) (namespaced bool, ok bool)
}

// BuildObjects projects a slice of live unstructured objects into the per-relation
// closure.Object set the engine consumes. It is PURE: no client, no I/O, no cluster.
// It (1) builds a name→uid index over the input, (2) extracts the four relations from
// each object's unstructured map using the same field paths as internal/scenario, and
// (3) resolves cross-ref / owner uids through the index (leaving UID empty when the
// referent was not listed — crossRefMatches falls back to Kind/ns/name).
//
// It returns an error (fail-closed) only on a *malformed* object — a missing
// kind/name, or a selector with an unrecognised matchExpressions operator — mirroring
// the loader's parse-boundary rejections (matchLabels invalid-operator, scenario.go:329).
// A GVK whose scope ScopeInfo cannot resolve is a caller-level fail-closed concern
// (slice 4), not a builder error.
func BuildObjects(objs []unstructured.Unstructured, scope ScopeInfo) ([]closure.Object, error)
```

### `internal/cluster` — the name→uid resolver (SLICE 1, this slice)
```go
// nameUIDIndex maps a (Kind, namespace, name) tuple to the real metadata.uid of a
// listed object, so a cross-ref / owner that names only kind+name can be resolved to
// the authoritative uid. It is built once over the BuildObjects input and consulted
// while extracting each object's relations. A miss leaves UID empty (the referent was
// not listed); crossRefMatches/OwnedChildren then fall back to Kind/ns/name.
type nameUIDIndex map[humanKey]string // humanKey = {Kind, Namespace, Name}

func newNameUIDIndex(objs []unstructured.Unstructured, scope ScopeInfo) (nameUIDIndex, error)
func (ix nameUIDIndex) uidFor(kind, namespace, name string) string // "" on miss
```
The resolver is unexported; it is exercised through `BuildObjects` (behaviour through
the public API, per the engineering default). `newNameUIDIndex` returns an error on a
malformed object (no kind/name) so `BuildObjects` fails closed at one boundary.

### `internal/cluster` — the live reader (SLICE 2, this slice)
```go
// Reader reads a live cluster READ-ONLY and returns a closure.State. It holds a
// discovery.DiscoveryInterface and a dynamic.Interface, both injectable so tests
// drive it with discovery/fake + dynamic/fake and no real cluster.
type Reader struct { /* discovery.DiscoveryInterface, dynamic.Interface */ }

// NewReader builds a Reader from a *rest.Config (resolved from kubeconfig/context in
// slice 3). It constructs the discovery + dynamic clients; the cfg is used ONLY to
// build clients and is never logged (it carries the bearer token / client cert).
func NewReader(cfg *rest.Config) (*Reader, error)

// newReader is the injectable test seam: it takes already-built clients.
func newReader(disc discovery.DiscoveryInterface, dyn dynamic.Interface) *Reader

// State discovers server resources (ServerGroupsAndResources), selects the relevant
// GVRs — the broad safe default (workloads, pods, services, configmaps, secrets,
// PVCs, PDBs, networkpolicies, HPAs, namespaces) plus discovered CRDs (any non-built-in
// group), intersected with what discovery reports — lists each via the dynamic client
// (verb "list" ONLY), records each GVK's namespaced/cluster scope into a ScopeInfo,
// calls BuildObjects, and returns closure.NewScanState(objs). FAIL-CLOSED on any
// discovery or list error: it returns an error, never a silently shrunk State.
func (r *Reader) State(ctx context.Context) (closure.State, error)
```

#### Discovery-backed `ScopeInfo` (SLICE 2)
`discoveryScope` (a `map[closure.GVK]bool`) implements `ScopeInfo` from the discovery
answer: each discovered (group, version, kind) → its `APIResource.Namespaced` flag. A
GVK absent from discovery reports `ok=false` so the builder/caller fails closed rather
than guess (guessing cluster-scoped would let a namespaced object escape containment).
`newDiscoveryScope(disc)` builds it and fails closed on a discovery error. This is the
live replacement for the loader's static `clusterScopedKinds` map.

#### Read-only RBAC (SLICE 2, security posture)
The least-privilege manifest the live read requires is `deploy/rbac/krsm-reader-clusterrole.yaml`:
a `ClusterRole` with verbs **`get`/`list`/`watch` only** (no wildcard verb, no write
verb) over the explicit resource set above, plus a `ClusterRoleBinding` to a dedicated
`ServiceAccount`. Enforced read-only at the code level by `TestReaderUsesOnlyReadVerbs`
(runtime action-tracker inspection) and `TestPackageInvokesNoWriteVerbs` (static source
guard forbidding any dynamic mutation call site).

### CLI live path (SLICE 3 — deferred, not implemented now; do NOT touch `cmd/krsm/main.go`)
`krsm check [--context X] [--kubeconfig P] <verb> <Kind/name> [-n ns]` selects the live
path when cluster flags are present; a positional dir keeps the existing scenario path.
Builds the live `State`, runs the **shared** `closure.Safe`/`mode.Apply`/`writeReport`.

## Data model

### What the builder reads from each unstructured object (same paths as the loader)
| Relation | Field path(s) read | closure.Object field | Loader reference |
|---|---|---|---|
| identity | `apiVersion`, `kind`, `metadata.namespace`, `metadata.name`, **`metadata.uid`** | `Ref` (incl. real `UID`) | `parseCluster`, `gvkOf` |
| labels | `metadata.labels` | `Labels` | `rawMetadata.Labels` |
| ownerRefs (rel 1) | `metadata.ownerReferences[].{kind,name,**uid**}` | `Owners` (real `UID`) | `parseCluster` owners loop |
| namespace (rel 2) | `metadata.namespace` + `ScopeInfo.Namespaced(gvk)` | `Ref.Namespace` (""=cluster) | `nsOf`/`clusterScopedKinds` |
| selector (rel 3) | `spec.selector` (Service flat map; workload/PDB `{matchLabels,matchExpressions}`), `spec.podSelector` (NetworkPolicy) | `Selector` | `selectorFrom`/`matchLabels` |
| cross-ref (rel 4) | `spec.volumes`/`spec.containers`/… AND `spec.template.spec.{volumes,containers,initContainers,ephemeralContainers,imagePullSecrets}`; `spec.scaleTargetRef`; `metadata.finalizers` | `CrossRefs`, `Finalizers` | `crossRefsFrom`/`podSpecCrossRefs` |

The volume/env sub-paths are read exactly as the loader's typed structs declare them:
- volumes: `configMap.name`, `secret.secretName`, `persistentVolumeClaim.claimName`,
  `projected.sources[].{configMap.name,secret.name}`;
- container `envFrom[].{configMapRef.name,secretRef.name}`;
- container `env[].valueFrom.{configMapKeyRef.name,secretKeyRef.name}`;
- `imagePullSecrets[].name`;
- `scaleTargetRef.{kind,name}` → `closure.RefScaleTarget`.

### Namespace resolution (the one semantic change from the loader)
- Loader: `nsOf(kind, ns)` uses the static `clusterScopedKinds` map; `ns==""`→`"default"`.
- Builder: `ScopeInfo.Namespaced(gvk)` decides. If **not namespaced** → `Ref.Namespace=""`
  (cluster-scoped, never counted in `NamespaceContents`, matches scope on ""). If
  **namespaced** → use `metadata.namespace`, defaulting `""`→`"default"` (matching the
  loader so the parity oracle holds). `[assumed: namespaced objects with an empty
  metadata.namespace default to "default", matching the loader's nsOf — in a live read
  the API server always populates it, so this only affects hand-built slice-1 fixtures.]`

### uid model (the fidelity improvement)
- `Ref.UID` = `metadata.uid` (a real uid string).
- `OwnerRef.UID` = `ownerReferences[].uid` (real). The owner edge is already a real uid
  on the child — exactly what `scanState.OwnedChildren` matches on — so **no name-based
  owner resolution and no `uidOf` synthesis**. This is strictly more faithful than the
  loader, which synthesises both sides.
- `CrossRef.Ref.UID` = `nameUIDIndex.uidFor(kind, ns, name)` of the referent; `""` when
  the referent was not listed (then `crossRefMatches` falls back to Kind/ns/name).

`[assumed: an ownerReferences entry always carries a real uid in a live read; a
hand-built slice-1 fixture that omits owner uid resolves UID="" and falls back to
name — but the parity test for scenario 01 sets real uids so the owner edge matches by
uid, proving the real-uid path.]`

## Behavior

### Slice-1 builder flow (per `BuildObjects` call)
1. Build `nameUIDIndex` over all input objects (Kind/ns/name → real uid). Error if any
   object lacks kind or name (fail-closed at the boundary).
2. For each object: resolve scope via `ScopeInfo` → namespace; read `metadata.uid` →
   `Ref.UID`; read labels; extract selector (kind-aware, same rules as `selectorFrom`);
   extract owners (real uids); extract cross-refs (volumes/env/imagePullSecrets/
   scaleTargetRef across top-level + `spec.template.spec`) resolving each referent's uid
   through the index; read finalizers.
3. Return `[]closure.Object`.

### Selector edge cases (must match `selectorFrom`/`matchLabels` exactly)
- **Service**: `spec.selector` is a flat map. Absent → `LabelSelector{}` (nil, binds
  nothing). Present (incl. `{}`) → `MatchLabels` set (the kind-aware `selectorBinds`
  decides empty-Service binds nothing).
- **NetworkPolicy**: `spec.podSelector.{matchLabels,matchExpressions}`. Absent → nil.
  Present-empty `{}` → non-nil empty map (binds all pods in ns — corpus #8).
- **Workload/PDB (default)**: `spec.selector.{matchLabels,matchExpressions}`.
- **matchExpressions**: each `{key,operator,values}`; an **unrecognised operator is
  rejected** (`BuildObjects` errors), mirroring `matchLabels` (scenario.go:329) — never
  silently dropped (a dropped binding is a missed escape).

### Parity (the slice-1 acceptance behaviour, the key test)
Hand-built unstructured fixtures equivalent to golden `01-memory-pressure-cascade`
(Deployment→ReplicaSet→3 Pods via real ownerReference uids, Service `spec.selector:
{app: web}`), fed to `BuildObjects`, then `closure.NewScanState(objs)` →
`closure.Closure`/`closure.Safe` with scenario 01's action (`delete deployment/web
cascade`) and scope (`Pod prod/web-1`), must produce the **same closure and escaping
set** as `closure/testdata/scenarios/01/expected.yaml`. The corpus is the parity oracle.

### Fail-closed (slices 2/4 — deferred)
Unresolvable target → existing `Safe` empty-closure deny. Unreadable/forbidden kind or
discovery failure → deny the whole check (never a partial snapshot). Not in slice 1.

## Components & responsibilities
- `internal/cluster/build.go` — `BuildObjects`, the four extractors ported to
  unstructured (`selectorFromUnstructured`, `crossRefsFromUnstructured`,
  `podSpecCrossRefs`, owner/finalizer reading), the GVK parse from `apiVersion`. **Pure,
  apimachinery-only deps. (SLICE 1)**
- `internal/cluster/index.go` — `nameUIDIndex` + `ScopeInfo`. **(SLICE 1)**
- `internal/cluster/build_test.go` — table tests per relation + the scenario-01 parity
  test. **(SLICE 1)**
- `internal/cluster/reader.go` — discovery + dynamic `Reader`, `NewReader`/`newReader`,
  the GVR selection (`selectTargets`), and the discovery-backed `discoveryScope`
  ScopeInfo (`scopeFromLists`/`newDiscoveryScope`). Read-only. **(SLICE 2, this slice)**
- `internal/cluster/reader_test.go` — discovery-scope mapping, Reader list→State,
  cluster-scoped mapping, fail-closed on discovery/list error, no-write assertions,
  `NewReader` construction. **(SLICE 2, this slice)**
- `deploy/rbac/krsm-reader-clusterrole.yaml` — least-privilege read-only RBAC
  (get/list/watch, no wildcards/writes). **(SLICE 2, this slice)**
- `cmd/krsm/main.go` — CLI live path. **(SLICE 3, deferred — DO NOT TOUCH now.)**
- `internal/archguard/archguard_test.go` — unchanged; the existing `TestClosureIsStdlibOnly`/
  `TestScopeIsStdlibOnly` are the guard that apimachinery does not leak into `closure`/`scope`.
- `go.mod` — adds `k8s.io/apimachinery` (slice 1 only needs `unstructured`).
  client-go/api are slice 2+.

## Alternatives considered
- **Share the extractors with `internal/scenario` instead of porting.** Rejected for
  this slice (plan decision 2): the loader reads typed YAML structs, the builder reads
  unstructured maps; a shared relation-projection helper is a later refactor. Parallel,
  not shared, now — keeps each path simple and the parity test honest.
- **Typed decode (`k8s.io/api`) for well-known kinds.** Rejected (plan open Q2, lean
  unstructured): unstructured keeps the extractor generic over CRDs and avoids pulling
  `k8s.io/api` into slice 1. Add it only if a typed decode is ever needed.
- **Static `clusterScopedKinds` map in the live path.** Rejected (plan decision 2/4):
  discovery is the live source of truth; the static map stays only in the loader (YAML
  can't carry scope). Slice 1 abstracts scope behind `ScopeInfo` so the builder never
  hard-codes it.
- **Build a shared-informer index now.** Rejected (plan decision 1): a one-shot
  `krsm check` gains nothing from informer warm-up/watch/resync; the pure builder is the
  reusable seam the webhook will later feed from caches.

## Trade-offs & risks
- **Optimises for parity** with the loader's relations (the highest risk, plan §Risks):
  same field paths, same kind-aware selector rules, same cross-ref kinds — validated
  against the corpus before any cluster is touched.
- **Cost:** apimachinery enters `go.mod` (larger dep tree, `govulncheck` surface — report,
  don't suppress). Confined to `internal/cluster`; archguard proves `closure`/`scope`
  stay stdlib-only.
- **nil-vs-present-empty selector** must survive the unstructured round-trip: a missing
  `spec.selector` is nil (binds nothing); a present `{}` is a non-nil empty map. The
  extractor distinguishes them via `NestedMap`'s `found` return, exactly as the loader
  distinguishes `json.RawMessage == nil`.

## Testability
Every behaviour is observed through `BuildObjects` (public) and the engine (`closure.Closure`/
`closure.Safe` over `NewScanState`):
- **Owner edge by real uid** — two objects, child carries `ownerReferences[].uid` ==
  parent `metadata.uid`; assert `OwnedChildren(parent)` returns the child.
- **Namespace containment** — namespaced objects appear in `NamespaceContents(ns)`; a
  cluster-scoped GVK (per `ScopeInfo`) resolves to `Namespace=""` and does not.
- **Selector** — Service flat map binds matching pod; NetworkPolicy `spec.podSelector`
  binds; present-empty `podSelector: {}` binds all; `matchExpressions` In/NotIn/Exists/
  DoesNotExist bind precisely; unrecognised operator → `BuildObjects` errors.
- **Cross-refs** — volume configMap/secret/PVC, projected sources, envFrom, env
  valueFrom, imagePullSecrets, and the same under `spec.template.spec`; `scaleTargetRef`
  (RefScaleTarget); finalizers populated; cross-ref uid resolved through the index and
  empty on a miss.
- **Parity** — scenario-01-equivalent fixtures → closure/escaping == `expected.yaml`.

## Open questions
1. `ScopeInfo` shape: a single `Namespaced(gvk) (bool, ok bool)` method (this design) vs
   a prebuilt `map[GVK]bool`. Lean interface so slice 2's discovery client and slice-1
   fakes both satisfy it; resolved here as the interface.
2. Whether `BuildObjects` should dedupe objects sharing a (Kind,ns,name) but differing in
   uid. `[assumed: a live list never returns duplicates; slice-1 fixtures won't either —
   defer dedup to slice 2 if observed.]`
3. client-go version pin and any `go.mod` Go-line bump — spiked in slice 2; slice 1 only
   needs `k8s.io/apimachinery` (no client-go).
