# Design: v0.2-cli-check
Status: approved · Date: 2026-06-20 · Slug: `v0.2-cli-check` · Plan: docs/plans/v0.2-cli-check.md

## Summary
Add a `krsm check <scenario-dir>` subcommand that loads a scenario's `cluster.yaml` / `request.yaml`
/ `scope.yaml`, runs `closure.Safe`, and prints a README-shaped `ACTION / SCOPE / CLOSURE / VERDICT`
report with a scriptable exit code. The YAML-to-`closure` loading logic that today lives test-only in
`closure/loader_test.go` moves into a new **`internal/scenario`** package so both the CLI and the
existing golden test use one loader, while `closure/`'s public API stays stdlib-only / YAML-free. This
completes v0.2's "each scenario doubles as a runnable demo" clause and makes the README real.

## Interfaces / contracts

### `internal/scenario` (new)
```go
package scenario

// Scenario is a runnable check input: the three things closure.Safe needs.
type Scenario struct {
    State  closure.State
    Action closure.Action
    Scope  []closure.ScopeRef
}

// Load reads cluster.yaml, request.yaml and scope.yaml from dir and builds a
// Scenario. It does NOT read expected.yaml (that is a test-only assertion
// artifact). Returns an error if a file is missing or malformed.
func Load(dir string) (*Scenario, error)
```
- The cluster/request/scope parsers, uid synthesis, and the `sigs.k8s.io/yaml` dependency move here
  (unexported). They are the current `loader_test.go` helpers, re-shaped to **return errors** instead
  of calling `t.Fatalf`. Parsing semantics are unchanged (e.g. `Cascade` defaults to `true` when
  absent; `Namespace` defaults to `default` for namespaced kinds; selector flattening per kind).
- `Load` reads exactly `dir/{cluster,request,scope}.yaml`; it imposes no convention on `dir` itself,
  so the CLI can pass any path and the golden test can pass `testdata/scenarios/<name>`.

### `cmd/krsm` — `check` subcommand
`run`'s signature gains a stderr stream so the demo keeps stdout clean and advisories go to stderr:
```go
// was: func run(args []string, out io.Writer) error
func run(args []string, stdout, stderr io.Writer) error
```
- `krsm check <dir>` → loads the scenario, runs `closure.Safe(sc.State, sc.Action, sc.Scope)`, writes
  the report to `stdout`, and signals the verdict through the **return + exit code** (below).
- Missing `<dir>` arg or load error → returns a normal wrapped error (usage/load failure).

### Exit-code contract (the scriptable part)
| Situation | `run` returns | Process exit |
|---|---|---|
| `Allow` | `nil` | `0` |
| `Warn` | `nil` (full WARN verdict + external detail to `stderr`) | `0` |
| `Block` | `errBlocked` (sentinel, no message) | `2` |
| usage error / load failure | wrapped `error` | `1` |
| `version` / `help` | `nil` | `0` |

`errBlocked` is a package sentinel that carries *status, not a message* — the human-readable Block
report is already on stdout. `main` maps it to exit 2 **without** printing the `krsm:` error prefix:
```go
err := run(os.Args[1:], os.Stdout, os.Stderr)
switch {
case err == nil:                 // exit 0
case errors.Is(err, errBlocked): os.Exit(2)
default: fmt.Fprintln(os.Stderr, "krsm:", err); os.Exit(1)
}
```

## Data model
- `scenario.Scenario{State, Action, Scope}` — the only exported type; the `raw*` YAML structs stay
  unexported in `internal/scenario`. No persistence; pure file-in / value-out.
- No change to any `closure` type. `Decision{Verdict, Reason, Closure, Escaping, External}` is consumed
  as-is for the report.

## Behavior

### Report format (written to stdout)
The factual report (`ACTION` / `SCOPE` / `CLOSURE`) always goes to **stdout**. The `VERDICT` line and
its `→` detail are routed by severity. Aligned labels, refs via `Ref.String()` (= `Kind/ns/name`),
reason from `Decision.Reason`:
```
ACTION   <verb> <Kind>/<ns>/<name>          (stdout)
SCOPE    <Kind>/<ns>/<name>, <Kind>/<ns>/<name>   (stdout)
CLOSURE  <ref>, <ref>, ...                  (stdout)
VERDICT  <icon> <ALLOW|WARN|BLOCK> — <reason>     (stream per verdict, below)
           → <ref>                          (escaping members for BLOCK; external effects for WARN)
```
- **Allow → stdout:** `VERDICT  ✅ ALLOW — closure within task scope` (no `→` lines). Exit 0.
- **Block → stdout:** `VERDICT  ❌ BLOCK — affected-resource closure escapes task scope:` followed by
  one `→ <ref>` line per `Decision.Escaping`. Exit 2. (The denial is the primary result the caller
  asked for, so it stays on stdout with the report.)
- **Warn → both streams (split by role):** stdout gets a self-contained WARN **stub**
  (`VERDICT  ⚠ WARN — <reason> (detail on stderr)`) so a reader of stdout still sees the verdict word;
  the full `VERDICT  ⚠ WARN — <reason>:` line plus one `→ <ref>` per `Decision.External` go to
  **stderr** (the cross-boundary detail needs live-cluster confirmation, so it is a diagnostic, not a
  result). Exit 0. *(Implementation note: an earlier draft put the verdict on stderr only and kept
  stdout purely factual; the shipped behavior adds the stdout stub so a piped stdout is not silent
  about a WARN. `TestCheckWarnGoesToStderr` pins both halves.)*
- The README's editorial annotations (e.g. `(live replica)`) are dropped — the engine does not produce
  them. Tests assert on the verdict word + ref set + stream, not on the emoji, to stay robust.
- **`--plain` flag.** `krsm check --plain <dir>` suppresses the verdict emoji (`❌`/`⚠`/`✅`), emitting
  ASCII-only output for CI logs / non-UTF8 terminals; the verdict word, refs, streams, and exit codes
  are unchanged. Flags must precede `<dir>` (Go `flag` semantics); a stray positional after `<dir>` is
  a usage error (exit 1), not a silently-dropped token.

### Edge cases
- Unknown/absent target (scenario 11) → `Safe` already returns `Block` with the fail-closed reason; the
  report prints it and exits 2, same as any Block. No special-casing in the CLI.
- Empty closure / no escapes → `Allow`.
- `check` with no dir arg → usage error (exit 1).

## Components & responsibilities
- **`internal/scenario/scenario.go`** (new) — owns YAML→`closure` loading; the only place
  `sigs.k8s.io/yaml` is imported outside tests.
- **`internal/scenario/scenario_test.go`** (new) — `Load` over the 12 dirs returns usable inputs;
  error on missing dir / bad YAML.
- **Golden/bound tests → `internal/scenario` (`golden_test.go`, `package scenario`).** *Build-time
  refinement to plan Q2:* an **internal** `closure` test cannot import `scenario` (which imports
  `closure`) without an import cycle, so `TestScenarios` and `TestClosureBoundedByInventory` move into
  `package scenario`. They drive `closure` through its **exported** API (`Safe`, `Decision`,
  `Ref.String()`), and the bound test gets `|R|` from this package's own `parseCluster` (no `closure`
  internals leak). The 12 golden assertions remain the behavior guard proving the extraction changed
  nothing. `closure/loader_test.go` is **removed** (its parsers now live in `scenario.go`).
- **`closure/closure_test.go`** (slimmed) — keeps only `TestClosureTerminatesOnCyclicOwners`, a pure
  property test built from exported constructors with no scenario loader, so `package closure` keeps a
  self-contained closure property test and imports nothing.
- **`cmd/krsm/main.go`** — `run` gains the stderr param + `check` case + a `writeReport(w, sc, dec)`
  formatter; adds the `errBlocked` sentinel and the exit-code mapping in `main`; usage banner lists
  `check`.
- **`cmd/krsm/main_test.go`** — updated `run(...)` call sites (extra buffer) + `check` tests.
- **README.md** / **CHANGELOG.md** — slice 3: demo block made real with the actual command; changelog
  entry.

## Alternatives considered
- **Loader in the public `closure` package** — rejected: leaks YAML/`raw*` types into the stdlib-only
  embeddable SDK, the exact violation that got the parity harness ejected.
- **Loader inside `cmd/krsm`** — rejected: not reusable by the golden test or by v0.4 live reads;
  forces a duplicate parser.
- **`run` returns `(int, error)`** — rejected: abandons the `error` idiom for `version`/`help` and
  churns more; a `Block` sentinel + stderr stream is smaller and idiomatic.
- **Block as a normal `error`** — rejected: `main` would print a scary `krsm: ...` line for an expected
  Block whose report is already nicely on stdout.
- **`--verify` against expected.yaml in the CLI** — deferred (plan non-goal); expected parsing stays
  test-only.
- **Keeping `run(args, out)` (no stderr)** — rejected: the Warn advisory and clean-stdout demo want a
  separate stream; adding the param touches only our three tests.

## Trade-offs & risks
- **Optimizes for** SDK purity + one shared loader + a scriptable demo; **costs** a mechanical refactor
  of `loader_test.go` and a one-line signature change rippling through `main_test.go`.
- **Risk:** the loader move silently changes parsing behavior. *Mitigation:* the 12 golden scenarios
  must stay green through the refactor (behavior-preserving by construction).
- **Risk:** exit-code semantics surprise scripts. *Mitigation:* documented table above; `Block`=2 is
  distinct from usage/error=1 so callers can tell "ran, denied" from "couldn't run".

## Testability
- **`scenario.Load`** — table test over the 12 dirs asserts a non-nil `State`/`Action`/`Scope`; a
  missing-dir and a malformed-YAML case assert errors.
- **`run` + `check`** — call `run([]string{"check", dir}, &out, &errBuf)` for representative scenarios:
  01 (Block: stdout contains `BLOCK` + both escaping pod refs, returns `errBlocked`), an Allow case
  (returns nil, `ALLOW` on stdout, no `→`), 09 (Warn: returns nil, `WARN` + external ref appear on
  **stderr** while stdout holds only `ACTION`/`SCOPE`/`CLOSURE`), 11 (Block fail-closed: `errBlocked`,
  reason mentions fail-closed). Exit-code mapping is
  observed at the `run` boundary via the returned sentinel (the `os.Exit` in `main` is the thin
  untested shell).
- **Regression** — the unchanged `closure.TestScenarios` over all 12 scenarios proves the loader
  extraction preserved behavior; full gate (`gofmt`, `go vet`, `staticcheck`, `golangci-lint`,
  `go test -race ./...`) green.

## Open questions
None. All resolved (approved 2026-06-20): loader → `internal/scenario`; golden test stays in
`package closure`; dir-only input; **Block → exit 2**, Allow/Warn → exit 0; output drops the editorial
annotations; ALLOW/BLOCK verdicts stay on stdout with the report. The WARN **external detail** goes to
stderr; stdout additionally carries a WARN **stub** so a piped stdout reports the verdict (refined
during the build — see the Warn bullet under *Report format*). A post-approval addition: the `--plain`
flag (ASCII-only, no emoji) and rejection of a stray positional after `<dir>`.
