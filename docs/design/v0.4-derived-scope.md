# Design: v0.4-derived-scope
Status: draft · Date: 2026-06-22 · Slug: `v0.4-derived-scope` · Plan: docs/plans/v0.4-derived-scope.md

## Summary
Make KRSM useful with **no declared scope** (ADR-0011). Two new scope dimensions —
`namespace` (pure-`Ref`) and `ownership` (root + transitive owned, state-dependent) —
join `resource`/`selector` in the engine; a pure, stdlib-only `scope.Derive(target)`
synthesizes a **Level-0** predicate (the ownership tree of the request target, *alone*);
the loader/CLI use it when a scenario has neither `scope.yaml` nor `taskcontract.yaml`;
and a `scope.Mode` (`ModeAudit` default / `ModeEnforce`) transform sits **above** the
unchanged `closure.Safe`, downgrading a scope-escape `Block→Warn` in audit. The
`ownership` dimension forces the unexported `matchScope` label-lookup seam
(`closure/safe.go:74`) to widen into a narrow **resolver** the `Safe` caller builds from
the `State` it already holds — with ownership-subtree membership computed by the *same*
`s.OwnedChildren` owner→child walk `closure.Closure` uses (`closure/closure.go:40`), so
scope-ownership and closure-ownership agree by construction. `closure.Safe`'s signature
and the closure walk do not change; `closure` and `scope` stay stdlib-only.

## Interfaces / contracts

### `closure` — two new dimensions (additive to `ScopeClause`)
```go
const (
    DimResource  ScopeDim = "resource"  // existing
    DimSelector  ScopeDim = "selector"  // existing
    DimNamespace ScopeDim = "namespace" // new: every resource in a namespace
    DimOwnership ScopeDim = "ownership" // new: a root + everything it transitively owns
)

// ScopeClause gains Root for the ownership dimension; every other field is as today.
type ScopeClause struct {
    Dim       ScopeDim
    GVK       GVK
    Namespace string
    Name      string        // DimResource only
    Selector  LabelSelector // DimSelector only
    Root      Ref           // DimOwnership only: the subtree root (the task target)
}

// NamespaceClause builds a DimNamespace clause: every candidate in ns (GVK optional).
func NamespaceClause(gvk GVK, namespace string) ScopeClause

// OwnershipClause builds a DimOwnership clause rooted at root: root itself plus every
// resource transitively owned by it (owner→child by uid). State-dependent.
func OwnershipClause(root Ref) ScopeClause
```
- `Safe(s State, a Action, scope []ScopeClause) Decision` — **signature unchanged**
  (`closure/safe.go:17`). The two new dimensions are handled inside the unexported
  `matchScope`; the public surface gains only the two constructors, the two `ScopeDim`
  constants, and the `Root` field.
- `ScopeClause.Validate()` (`closure/model.go:169`) extends to the new dims (rules in
  Data model).

### `closure` — the `matchScope` seam widens (internal)
```go
// scopeResolver answers the state-dependent questions matchScope's dimensions ask,
// over the same snapshot the closure was computed on. It replaces the bare labelsOf
// parameter (closure/safe.go:74): selector needs labels, ownership needs subtree
// membership, and a future reference dim gets a method here rather than another
// positional func — the StateResolver evolution ADR-0008's alternatives anticipated.
type scopeResolver interface {
    labels(Ref) (map[string]string, bool)   // DimSelector
    ownedSubtree(root Ref) map[string]bool   // DimOwnership: keys are Ref.key(), incl. root
}

func matchScope(r Ref, scope []ScopeClause, res scopeResolver) bool
```
- `Safe` builds the concrete resolver from the `State` it already holds (replacing the
  inline `labels` closure at `closure/safe.go:33`). `labels` delegates to `s.Get`;
  `ownedSubtree` BFS-walks `s.OwnedChildren` from `root` with a visited-set guard and
  **memoizes** the result per `root.key()` (so an N-member closure costs one walk per
  distinct root, not N).
- `matchScope` stays unexported; the change is internal. `labelsOf` (`safe.go:74`) is
  removed in favour of the interface.

### `scope` — the Level-0 synthesizer + provenance + mode (public, stdlib-only)
```go
// Provenance records how a ScopePredicate's scope was obtained, for the verdict report.
type Provenance string
const (
    ProvenanceContract     Provenance = "contract"               // Compile (ADR-0009)
    ProvenanceDerivedOwner Provenance = "derived:ownership-tree" // Derive, Level 0
)

// ScopePredicate gains Provenance (Clauses/MaxSeverity unchanged, ADR-0009).
type ScopePredicate struct {
    Clauses     []closure.ScopeClause
    MaxSeverity Severity
    Provenance  Provenance
}

// Derive synthesizes a Level-0 predicate from an action target with no declared scope:
// a single ownership clause rooted at the target (the target + everything it owns).
// Deliberately NOT OR'd with a namespace clause — under the union semantics of
// C ⊆ scope that would re-admit same-namespace collateral and defeat the default
// (ADR-0011). Pure: the state-dependence lives in the engine, so Derive needs no State.
func Derive(target closure.Ref) ScopePredicate

// Mode governs how a scope-escape verdict is reported (ADR-0011). Audit is the install
// default: a derived/declared escape is surfaced as Warn (never a deny) so a day-0
// false positive does not get KRSM uninstalled; Enforce is opt-in.
type Mode string
const (
    ModeAudit   Mode = "audit"   // default
    ModeEnforce Mode = "enforce"
)

// Apply maps a decision for the given mode. In ModeAudit a scope-escape Block
// (len(Escaping) > 0) is downgraded to Warn, preserving Escaping/Closure for the
// report; a fail-closed Block (empty Escaping — closure uncomputable, DESIGN §5) is
// NOT softened. ModeEnforce returns the decision unchanged.
func (m Mode) Apply(d closure.Decision) closure.Decision
```
- `Compile` (`scope/scope.go:92`) sets `Provenance: ProvenanceContract` on its result;
  signature otherwise unchanged.

### `internal/scenario` + `cmd/krsm`
- `Scenario` gains `ScopeSource string` (human provenance for the SCOPE line, e.g.
  `derived (ownership-tree)`, `taskcontract.yaml`, `scope.yaml`).
- `loadScope` (`internal/scenario/scenario.go:79`) gains a final branch: when **neither**
  `taskcontract.yaml` **nor** `scope.yaml` exists, call `scope.Derive(action.Target)` and
  use its clauses (so the loader needs the parsed `Action` — `Load` already has it before
  `loadScope`). The two `errors.Is(err, os.ErrNotExist)` checks distinguish "no file" from
  a read error.
- `krsm check` gains `--mode audit|enforce` (default `audit`); `runCheck`
  (`cmd/krsm/main.go:99`) applies `mode.Apply` to the `Safe` result before
  `writeReport`, and the exit-status decision (`main.go:126`) acts on the *applied*
  verdict. The SCOPE line (`scopeStr`, `main.go:197`) renders the new dims
  (`ns:prod/*`, `owns:Deployment.apps/prod/web`) and prints `ScopeSource`.

## Data model

### Clause shapes & `Validate` rules
| Dim | Meaningful fields | `Validate` requires |
|---|---|---|
| `resource` (or `""`) | `GVK`, `Namespace`, `Name` | no `Selector`, no `Root` |
| `selector` | `GVK`, `Namespace`, `Selector` | no `Name`, no `Root` (empty selector stays a fail-safe match-nothing) |
| `namespace` | `GVK` (optional), `Namespace` | non-empty `Namespace`; no `Name`, `Selector`, `Root` |
| `ownership` | `Root` | `Root` has `GVK.Kind` and `Name`; no `Name`, `Selector`, clause-level `GVK`/`Namespace` (identity lives on `Root`) |
| _other_ | — | rejected (unknown dimension, fail-closed) |

`Validate` keeps its existing philosophy (`closure/model.go:169`): a field that a
dimension would silently ignore is a hard error, so a mis-built clause fails loudly at
load (`parseScope`/`Compile`) rather than misbehaving in the engine.

### `scope.yaml` / `taskcontract.yaml` wire forms (loader)
```yaml
# scope.yaml — explicit namespace / ownership clauses (for tests & power users)
scope:
  - dim: namespace
    namespace: prod
  - dim: ownership
    root: {group: apps, version: v1, kind: Deployment, namespace: prod, name: web}
```
The loader resolves `root`'s synthetic uid via the existing `uidOf` (as it does for the
action target, `internal/scenario/scenario.go:465`) so `ownedSubtree` membership matches
by uid like the closure does.

### Derived Level-0 (no wire form)
`scope.Derive(target)` → `ScopePredicate{Clauses: []closure.ScopeClause{
closure.OwnershipClause(target)}, Provenance: ProvenanceDerivedOwner}`.

## Behavior

### `matchScope(r, scope, res)` — OR across clauses, gate per dimension
The single top-level `gates` call (`closure/safe.go:91`) moves **into** the dimension
switch, because ownership is not GVK/namespace-gated:
- **`resource`** (and empty `Dim`): `gates(sc, r)` then `path.Match(sc.Name, r.Name)` — unchanged (`safe.go:96`).
- **`selector`**: `gates(sc, r)` then `matchSelectorClause` over `res.labels(r)` — unchanged (`safe.go:99`).
- **`namespace`**: `gates(sc, r)` *is* the membership test (GVK optional, `sc.Namespace == r.Namespace`) → in scope. No name/selector.
- **`ownership`**: `res.ownedSubtree(sc.Root)[r.key()]` — pure subtree membership, **no gate** (a child may differ in kind/namespace from the root).
- **unknown**: skip (covers nothing — fail-closed defence-in-depth, as today `safe.go:103`).

### `ownedSubtree(root)` — the same walk as the closure
BFS from `root` following `s.OwnedChildren` (`closure/state.go:73`) with a visited-set
guard, collecting `Ref.key()` for the root and every transitively owned resource.
Identical traversal to `closure.Closure`'s cascade step (`closure/closure.go:40`), so
"owned" means the same thing on both sides of `C ⊆ scope(T)`. Finite and cycle-safe:
each resource is visited once, so `|subtree| ≤ |R|`. Memoized per `root.key()`.

### Level-0 verdict (the zero-config flow)
`krsm check <dir>` with no scope file: `Load` derives `OwnershipClause(target)`; `Safe`
computes the closure; every member that is **not** in the target's ownership subtree
escapes. For scenario `01` (`delete deployment web`): the ReplicaSet and Pods are in the
tree (owned) → in scope; the **Service** is in the closure (selector binding,
`closure/closure.go:81`) but is *not* owned by the Deployment → it escapes → `Block`
(`enforce`) or `Warn` (`audit`). This is the "aha": the default flags the broken Service
with no contract written.

### Mode application (above `Safe`)
`runCheck`: `dec := mode.Apply(closure.Safe(...))`. In `audit`, a scope-escape `Block`
becomes `Warn` (exit 0, escaping set still printed so the operator sees what *would*
block); a fail-closed `Block` (`len(Escaping)==0`) stays `Block` (exit 2). In `enforce`,
unchanged. The report distinguishes the audit downgrade in its verdict line.

### Edge cases
- **Target with no children** (e.g. a bare ConfigMap): subtree = `{target}`; its consumers
  escape → correctly flagged (deleting a mounted ConfigMap affects pods).
- **Cross-namespace owned child**: still in the subtree (membership is by uid, not gate),
  so a legitimate cross-namespace ownership relationship is in scope; an *un-owned*
  cross-namespace member escapes (as it should).
- **Cyclic owners**: visited-set guard terminates; the cycle's members are all in the
  subtree.
- **Empty/`""` namespace clause**: rejected by `Validate` (a "namespace" clause must name
  a namespace; cluster-scoped scoping is the `resource` dim's job).

## Components & responsibilities
- **`closure/model.go`** — `DimNamespace`/`DimOwnership` constants; `ScopeClause.Root`;
  `NamespaceClause`/`OwnershipClause` constructors; extend `Validate`.
- **`closure/safe.go`** — replace `labelsOf` (`:74`) with `scopeResolver`; `Safe` builds
  the concrete resolver (memoized `ownedSubtree`) over its `State` (`:33`); restructure
  `matchScope` to gate per dimension and add the `namespace`/`ownership` branches.
  `Safe`'s signature and the escape/precedence logic are untouched.
- **`scope/scope.go`** — `Provenance`, `ScopePredicate.Provenance`, `Derive`, `Mode` +
  `Apply`; `Compile` stamps `ProvenanceContract`. Imports only `closure` + stdlib
  (`internal/archguard` `TestScopeIsStdlibOnly`).
- **`internal/scenario/scenario.go`** — `parseScope` learns `dim: namespace`/`ownership`
  (root ref + uid); `loadScope` derives when no scope file; `Scenario.ScopeSource`.
- **`cmd/krsm/main.go`** — `--mode` flag; `mode.Apply` before `writeReport`; `scopeStr`
  renders the new dims; SCOPE line shows `ScopeSource`.
- **Goldens** — explicit-`namespace` and explicit-`ownership` scenarios; a **no-scope-file**
  scenario proving Level-0 derivation; an audit-vs-enforce CLI/golden assertion.

## Alternatives considered
- **Pre-expand the ownership tree into resource clauses inside `Derive`** — rejected:
  `Derive` would need `State` (losing the pure, embeddable, stdlib-only property the
  other `scope` APIs have) and DESIGN §6 commits to an ownership *dimension*, not a
  flattened set. Keeping the state-dependence in `matchScope` mirrors `selector`.
- **OR a namespace-containment clause into the Level-0 default** (the first ADR-0011
  draft) — rejected during this design: under union semantics it re-admits the
  same-namespace collateral the tree is meant to flag (it would Allow scenario `01`), and
  it is redundant — a cross-namespace member is already outside the tree. The default is
  the ownership tree alone; the `namespace` dim remains for explicit/template use.
- **Pass full `State` into `matchScope`** — rejected (as ADR-0008 did for selector):
  couples scope matching to the whole `State` surface and makes unit tests build a full
  `State`. The narrow `scopeResolver` exposes exactly the two questions the dimensions
  ask and is trivially fakeable.
- **Keep `labelsOf` and add a second positional func for ownership** — rejected: the
  "accumulating function parameters" smell the v0.3 review flagged; one small interface
  with room for the future `reference` dim is cleaner.
- **Apply mode inside `Safe`** — rejected: mode is a deployment concern and ADR-0011 fixes
  `Safe` as unchanged; `Mode.Apply` above the engine lets the v0.5 webhook reuse it.
- **Lazy per-candidate `inSubtree(root, r)`** — rejected: re-walks the subtree per member;
  memoized `ownedSubtree(root)` is one O(tree) walk per distinct root.

## Trade-offs & risks
- **Optimises for** install-and-useful (zero-config verdict), formal fidelity
  (scope-ownership ≡ closure-ownership by shared walk), and an unchanged engine API.
- **Costs** an internal `matchScope` seam change (`labelsOf`→`scopeResolver`) rippling
  through `Safe`'s adapter and the `closure` selector unit test; a new `ScopeClause` field
  and two constructors; a richer loader + CLI.
- **Risk — resolver churns again when `reference` lands.** Mitigated: the interface is
  unexported and already shaped for "one method per state-need," so adding `reference` is
  a method, not a re-architecture.
- **Risk — golden regression.** Mitigated: resource/selector branches are byte-for-byte
  the old behaviour (only the gate moves inside the switch); all 22 existing goldens must
  stay green; the namespace/ownership branches are new code paths exercised by new
  goldens.
- **Risk — derived default over-flags** legitimate non-owned references. Mitigated: the
  audit default makes over-flagging visible, not disruptive; the cross-boundary allowlist
  is deferred to v0.5.

## Testability
Observed through public APIs and the golden corpus — never unexported helpers.
- **`closure` (package-internal unit table)** via `NewScanState` + `ScopeClause` values +
  `Safe`, asserted through `Decision.Escaping`:
  - `namespace` dim: a candidate in/out of the named namespace; GVK gate excludes wrong kind.
  - `ownership` dim: root in scope; direct child, grandchild in; sibling/unrelated out;
    a non-owned same-namespace resource (the scenario-`01` Service) out; cyclic owners
    terminate and are in-subtree.
  - resource/selector dims unchanged (regression of the moved gate).
- **`scope.Derive`** (package `scope`): `Derive(target)` → exactly one
  `OwnershipClause(target)`, `Provenance == ProvenanceDerivedOwner`, no namespace clause.
- **`scope.Mode.Apply`** (package `scope`): audit downgrades an escape `Block`
  (`len(Escaping) > 0`) → `Warn`, preserves `Escaping`; audit leaves a fail-closed `Block`
  (`len(Escaping) == 0`) as `Block`; enforce returns the decision unchanged.
- **Loader/golden:** an explicit `dim: namespace` and a `dim: ownership` scenario; a
  **no-scope-file** scenario whose `expected.yaml` pins the scenario-`01`-style verdict
  (Service escapes the derived tree); the negative `loadError` convention (ADR-0010) is
  untouched.
- **CLI (`cmd/krsm` test):** `--mode audit` renders an escape as `WARN` (exit 0) with the
  escaping detail; `--mode enforce` renders `BLOCK` (exit 2); the SCOPE line shows the
  derived provenance and the new dim rendering.
- **`archguard`:** `closure` and `scope` stay stdlib-only.
- **Full gate** `make check` green; the 22 existing goldens unchanged.
