# Test list: v0.2-cli-check  (source: docs/design/v0.2-cli-check.md)
Ordered as vertical slices — implement one before writing the next (red → green → refactor).

## Slice 1 — extract the loader (behavior-preserving)
1. [ ] `scenario.Load(dir)` loads scenario 01 and returns a usable `{State, Action, Scope}` such that
       `closure.Safe` on them yields a `Block` (proves the extracted parsers build correct inputs).
2. [ ] `closure.TestScenarios` stays green over all 12 scenarios when re-pointed to `scenario.Load`
       (regression guard: the extraction changed no parsing behavior).
3. [ ] `scenario.Load` returns an error for a missing directory and for malformed YAML.

## Slice 2 — `krsm check` end-to-end
4. [ ] `run(["check", <01-dir>], stdout, stderr)` prints `BLOCK` + both escaping pod refs to **stdout**
       and returns the `errBlocked` sentinel (→ exit 2).
5. [ ] `run(["check", <allow-dir>], …)` prints `ALLOW` to **stdout** with no `→` lines and returns nil
       (→ exit 0).
6. [ ] `run(["check", <09-warn-dir>], …)` writes the `WARN` verdict + external ref to **stderr** while
       **stdout** holds only `ACTION`/`SCOPE`/`CLOSURE`; returns nil (→ exit 0).
7. [ ] `run(["check", <11-dir>], …)` returns `errBlocked` and its `BLOCK` reason mentions fail-closed.
8. [ ] `run(["check"], …)` with no dir argument returns a non-`errBlocked` usage error (→ exit 1).
9. [ ] The report for a representative scenario shows `ACTION` (verb + target), `SCOPE` clauses, and a
       sorted `CLOSURE` ref list on stdout.

## Slice 3 — polish
10. [ ] Existing `version`/`help`/unknown-command behavior still holds under the new
        `run(args, stdout, stderr)` signature (updated call sites), and the usage banner lists `check`.

## Notes for the build
- `Decision.Reason` is empty for `Allow`; the ALLOW verdict line uses a CLI literal
  (`closure within task scope`); BLOCK/WARN lines use `Decision.Reason`.
- Block exit code = 2; usage/load error = 1; Allow/Warn = 0.
- Gate to keep green throughout: `gofmt`, `go vet`, `staticcheck`, `golangci-lint`, `go test -race ./...`.
- Git guardrail: stage at most; the user commits.
