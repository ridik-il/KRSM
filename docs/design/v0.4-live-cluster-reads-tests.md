# Test list: v0.4-live-cluster-reads  (source: docs/design/v0.4-live-cluster-reads.md)
Ordered as vertical slices — implement one before writing the next. SLICE 1 only.
All behaviour is observed through `cluster.BuildObjects` + the engine
(`closure.Closure`/`closure.Safe` over `closure.NewScanState`). Hand-built
`unstructured.Unstructured` fixtures; no cluster, no client.

1. [ ] BuildObjects projects identity (GVK from apiVersion, namespace, name, real metadata.uid) into closure.Object.Ref — Get(ref) returns it.
2. [ ] Owner edge by REAL uid: a child whose ownerReferences[].uid equals the parent's metadata.uid is returned by OwnedChildren(parent); no uidOf synthesis.
3. [ ] Namespace containment: a namespaced object appears in NamespaceContents(ns); a GVK reported cluster-scoped by ScopeInfo resolves to Namespace="" and does NOT appear.
4. [ ] Selector — Service flat spec.selector binds a matching pod (PodsSelectedBy / SelectorsTargeting); absent selector binds nothing; present-empty {} binds nothing (Service rule).
5. [ ] Selector — NetworkPolicy spec.podSelector.matchLabels binds; present-empty podSelector: {} binds ALL pods in ns (corpus #8).
6. [ ] Selector — matchExpressions (In/NotIn/Exists/DoesNotExist) bind precisely; an unrecognised operator makes BuildObjects return an error (fail-closed at the parse boundary).
7. [ ] Cross-ref — pod-template volumes (configMap/secret/persistentVolumeClaim/projected sources) under spec.template.spec.volumes produce RefVolume CrossRefs; Consumers(target) returns the workload.
8. [ ] Cross-ref — container envFrom (configMapRef/secretRef) and env[].valueFrom (configMapKeyRef/secretKeyRef) across containers/initContainers/ephemeralContainers produce RefEnvFrom/RefEnv CrossRefs.
9. [ ] Cross-ref — imagePullSecrets produce RefImagePullSecret; bare-Pod top-level spec.volumes/spec.containers are read as well as spec.template.spec.
10. [ ] Cross-ref — HPA spec.scaleTargetRef produces a RefScaleTarget CrossRef; ControllersTargeting(workload) returns the HPA.
11. [ ] Finalizers — metadata.finalizers populate Object.Finalizers verbatim.
12. [ ] Cross-ref uid resolution — a referent that IS listed gets its real uid via the name→uid index; a referent NOT listed leaves UID empty and still matches via Kind/ns/name fallback.
13. [ ] Malformed object — an object missing kind or name makes BuildObjects return an error (fail-closed boundary).
