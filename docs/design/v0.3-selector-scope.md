# Design: v0.3-selector-scope
Status: draft · Date: 2026-06-21 · Slug: `v0.3-selector-scope` · Plan: docs/plans/v0.3-selector-scope.md

## Summary
Give the **scope** side of `C(S,A) ⊆ scope(T)` the label-selector precision the **closure** side
already has (ADR-0005). Today a scope clause is flat identity — `ScopeRef{GVK, Namespace,
Name-glob}` matched by `matchScope` against a bare `Ref` (`closure/safe.go:62`) — so it cannot
express "the Pods matching `app=web`", and a precise closure tested against an imprecise scope
yields avoidable false **Block**s.

This slice replaces `ScopeRef` with a **dimension-typed `ScopeClause`** (DESIGN §6) carrying two
dimensions: `resource` (today's identity clause, behaviour unchanged) and `selector` (new,
state-dependent, reusing `closure.LabelSelector`). `matchScope` becomes state-aware via a narrow
label-lookup the `Safe` caller already has. A new golden scenario (`20-scope-selector-precision`)
is wrongly Blocked today and classifies `Allow` once selector scope exists. The full
`TaskContract`→`ScopePredicate` compiler and the other dimensions remain later-v0.3 work.

## Interfaces / contracts

### `closure` — scope clause becomes dimension-typed (replaces `ScopeRef`)
```go
// ScopeDim is the relation dimension a scope clause authorises along (DESIGN §6).
type ScopeDim string

const (
    DimResource ScopeDim = "resource" // flat identity: GVK + namespace + name(glob)
    DimSelector ScopeDim = "selector" // pods/objects whose labels satisfy Selector
)

// ScopeClause is one allow-clause of a task's authorised scope. Exactly one
// dimension's fields are meaningful, chosen by Dim. An empty Dim is read as
// DimResource for backward compatibility with v0.1/v0.2 flat scopes.
type ScopeClause struct {
    Dim       ScopeDim
    GVK       GVK           // gates both dims (empty Group/Version ignored, as today)
    Namespace string        // gates both dims
    Name      string        // DimResource only: identity or path.Match glob
    Selector  LabelSelector // DimSelector only: matchLabels + matchExpressions
}
```
- `func Safe(s State, a Action, scope []ScopeClause) Decision` — the only signature change to a
  public function: the `scope` parameter retypes `[]ScopeRef` → `[]ScopeClause`.
- `ScopeRef` is **removed** (no deprecated alias). It is a pre-1.0 breaking change to the public
  `closure` SDK, made now while there are no external importers — identical rationale to
  ADR-0005's `Object.Selector` retype. Recorded in CHANGELOG; carried by a follow-on ADR
  (selector scope dimension, ADR-0005 follow-on).

### `closure` — `matchScope` gains label access via a narrow lookup
```go
// labelsOf resolves a candidate's labels from the same snapshot the closure was
// computed over. Returns (labels, true) when the ref is tracked; (nil, false)
// otherwise.
type labelsOf func(Ref) (map[string]string, bool)

func matchScope(r Ref, scope []ScopeClause, labels labelsOf) bool
```
- `Safe` supplies the lookup by adapting the `State` it already holds:
  `labels := func(r Ref) (map[string]string, bool) { o, ok := s.Get(r); if !ok { return nil, false }; return o.Labels, true }`.
- `matchScope` stays unexported; the signature change is internal.

### `internal/scenario` — loader learns the dimension-typed clause
- `parseScope([]byte) ([]closure.ScopeClause, error)` (was `[]closure.ScopeRef`).
- `Scenario.Scope` retypes to `[]closure.ScopeClause`. `cmd/krsm` passes it straight through to
  `Safe`, so the CLI is otherwise untouched.

## Data model

### `scope.yaml` clause shape (corpus + future TaskContract precursor)
```yaml
scope:
  - dim: resource              # optional; defaults to resource when omitted
    kind: Deployment
    namespace: prod
    name: frontend             # exact or glob (path.Match), as today
  - dim: selector              # new
    kind: Pod                  # gvk gate (Group/Version optional)
    namespace: prod
    matchExpressions:          # matchLabels also accepted; both AND-ed (LabelSelector)
      - {key: app, operator: In, values: [web]}
```
- Loader raw struct gains `Dim`, `MatchLabels`, `MatchExpressions`; the selector is built with the
  **same** raw→`closure.LabelSelector` conversion the cluster loader already uses for
  `Object.Selector`, so there is one selector parse path.
- **Back-compat:** the 19 existing `scope.yaml` files have no `dim:` key → every clause loads as
  `DimResource` and matches exactly as before. The goldens stay byte-for-byte.

## Behavior

### `matchScope(r, scope, labels)` — clause-by-clause, OR across clauses
A member `r` is in scope iff **some** clause matches it:

- **`DimResource` (and empty `Dim`):** unchanged from `closure/safe.go:62` — `GVK.Kind` (and
  `Group`/`Version` when set) and `Namespace` must agree, then `path.Match(Name, r.Name)`.
- **`DimSelector`:**
  1. gate on `GVK`/`Namespace` exactly as the resource dim (so a `Pod` selector clause never
     matches a `Service`);
  2. if `Selector` is **empty or nil** → **match nothing** (fail-safe: an empty selector must
     never silently authorise the whole namespace);
  3. resolve `labels(r)`; if the candidate is **not tracked** → **match nothing** (fail-closed,
     DESIGN §5 — never grant scope to a resource we cannot see);
  4. otherwise match iff `clause.Selector.Matches(labels)` — the existing four-operator,
     absence-sensitive evaluation (`closure/selector.go`).

Snapshot consistency: `labels` is backed by the same `State` instance `Safe` used to compute the
closure, so a selector clause "the `app=web` Pods" expands over the same world the closure walked
(DESIGN §6).

### Effect on the verdict
`Safe`'s structure is unchanged: for each closure member, `matchScope` decides escape; precedence
stays `Block > Warn > Allow`. The only difference is that selector clauses can now cover members
that previously escaped, turning avoidable Blocks into Allow.

### Edge cases
- Selector clause with both `matchLabels` and `matchExpressions` → AND-ed by `LabelSelector.Matches`.
- A `*`-name resource clause still works (over-grant remains the author's choice); the selector dim
  is the precise alternative, not a replacement.
- Cluster-scoped candidate (`Namespace == ""`) + selector clause with `Namespace: ""` → gated in;
  matches on its labels like any other. No Pod-only special-casing.

### Proof scenario `20-scope-selector-precision`
Demonstrates a false Block that is *only* fixable with selector scope.

- **cluster.yaml** (prod): `Deployment/frontend` labels `{app: web}`, `spec.selector.matchLabels
  {app: web}`; `Pod/frontend-aaa` `{app: web}`; `Pod/worker-bbb` `{app: web}` (a *different*
  workload's pod, no shared name stem with `frontend-aaa`); `Pod/db-ccc` `{app: db}` (decoy: not
  `app=web`). No Service/PDB/NetworkPolicy (keeps the closure to the Deployment + its selected
  pods).
- **request.yaml:** `verb: restart, target: Deployment/prod/frontend`. Per `classify` (Restart on a
  workload → `Disruptive`) and `Closure` step 3, this binds **all** `app=web` pods in the namespace
  via `PodsSelectedBy`→`PodsMatching` (ownership-independent, state.go:110): closure =
  `{Deployment/frontend, Pod/frontend-aaa, Pod/worker-bbb}`; `db-ccc` excluded.
- **scope.yaml:** `dim: resource` Deployment/frontend **+** `dim: selector` Pod/prod
  `matchExpressions: [{key: app, operator: In, values: [web]}]`.
- **expected.yaml:** `verdict: Allow`, closure = the three refs, `escaping: []`, `external: []`.
- **Why it is wrongly Blocked today:** `frontend-aaa` and `worker-bbb` share no non-`*` name-glob,
  so a flat-identity scope cannot cover both without `*` (which would also grant `db-ccc`,
  defeating precision). With the selector clause unparsed (today), those pods escape → Block. The
  `In [web]` selector covers exactly the two and nothing else → Allow. This ties the fix directly to
  ADR-0005's `matchExpressions`.

## Alternatives considered
- **Pass `State` into `matchScope`** (not a narrow `labelsOf`) — rejected: couples scope matching to
  the whole `State` interface and forces unit tests to build a full `State`; `matchScope` only needs
  labels-by-ref. The lookup is trivially fakeable and keeps the dependency surface minimal.
- **Pre-resolve every closure member's labels into a map in `Safe`** — rejected: redundant work for
  scopes with no selector clause (the common case) and an extra structure to thread; lazy lookup per
  selector-clause evaluation is simpler and only pays when a selector clause exists.
- **Optional `Selector` field on a still-flat `ScopeRef`** (no `Dim`) — rejected: DESIGN §6 already
  commits scope to dimension-typed clauses, and ROADMAP v0.3 enumerates the five dimensions. A flat
  field would be reworked into a discriminator the moment the compiler slice lands; the
  discriminated `ScopeClause` is barely larger now and is the design's committed trajectory.
- **Keep `ScopeRef` as a deprecated alias** — rejected: no external importers exist (pre-1.0); a
  clean break is honest and avoids two names for one concept (same call as ADR-0005).
- **Kind-aware empty-selector semantics in scope** (mirroring `selectorBinds`' "empty NetworkPolicy
  selector binds all") — rejected for *scope*: in a binding it is a faithful k8s rule; in an
  authorisation it is a silent over-grant. An empty scope selector matches nothing.

## Trade-offs & risks
- **Optimises for** closing the `C ⊆ scope(T)` precision asymmetry with the smallest faithful step,
  stdlib-only SDK purity (`internal/archguard`), and a corpus scenario that *proves* the fix.
- **Costs** a pre-1.0 public API break (`ScopeRef`→`ScopeClause`, `Safe` param) rippling through
  `internal/scenario` and the new unit test; a slightly richer loader.
- **Risk — golden regression.** Mitigation: the resource-dim path is byte-for-byte the old
  `matchScope`; existing `scope.yaml` (no `dim`) loads as `DimResource`; all 19 goldens must stay
  green through the change (behaviour-preserving by construction).
- **Risk — silent over-grant via empty selector.** Mitigation: empty/nil selector → match nothing;
  unresolvable labels → match nothing (both fail-safe), pinned by unit tests.
- **Risk — the scenario doesn't actually prove the fix.** Mitigation: design-check/TDD runs it
  pre-change (expect Block) and post-change (expect Allow); the two-name-stem construction makes the
  flat-scope impossibility concrete, not asserted.

## Testability
- **`matchScope`/`Safe` selector unit test** (`package closure`, no loader): table over
  (`ScopeClause`, candidate labels) →
  - `matchLabels` hit / miss; `matchExpressions` `In`/`NotIn`/`Exists`/`DoesNotExist` incl. absence;
  - GVK/namespace gating still excludes wrong-kind/wrong-ns candidates;
  - empty selector clause → escape (match nothing); untracked candidate → escape (fail-closed);
  - a resource-dim clause still matches by name/glob exactly as before.
  Built from `NewScanState` + `ScopeClause` values + `Safe`, observed via `Decision.Escaping`.
- **Golden scenario 20** — `Safe` over the loaded scenario yields `Allow` with the three-ref closure
  and empty `escaping`; asserted by the directory-driven `internal/scenario` golden test.
- **Regression** — the unchanged 19 goldens prove the resource-dim path and loader back-compat are
  untouched.
- **Full gate** — `make check` (`gofmt`, `go vet`, `staticcheck`, `golangci-lint`, `go test -race
  ./...`) green; `internal/archguard` confirms `closure` stays stdlib-only.

## Open questions
None blocking. Resolved here:
1. **`matchScope`↔`State` seam** → narrow `labelsOf func(Ref)(map[string]string,bool)` adapter over
   `s.Get`, supplied by `Safe`.
2. **Clause encoding** → single `ScopeClause` sum type with a `ScopeDim` tag (`resource`,
   `selector`); `[]ScopeRef` → `[]ScopeClause`; empty `Dim` reads as `resource` for back-compat.
3. **Empty/nil selector clause** → match nothing (fail-safe), not load-time rejection (keeps the
   loader lenient; the engine is the safety boundary).
4. **Selector clause kinds** → gated by the clause's `GVK` like resource clauses; no Pod-only rule.

Deferred to later v0.3 slices (explicit non-goals): the `TaskContract`→`ScopePredicate` compiler and
`scope/` package; the `ownership`/`namespace`/`reference` dimensions; inexpressible-scope (e.g.
set-difference) gap signalling.
