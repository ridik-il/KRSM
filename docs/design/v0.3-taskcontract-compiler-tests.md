# Test list: v0.3-taskcontract-compiler  (source: docs/design/v0.3-taskcontract-compiler.md)

Ordered as vertical slices — implement one before writing the next (red → green → refactor).
Every entry is observed through a **public interface**: `scope.Compile`, the `internal/scenario`
loader (`Load`), the golden corpus, or the `archguard` test. No test touches unexported helpers.

> **Build order (for `tdd`):** first green step is the `scope` package skeleton — the
> `TaskContract`/`Spec`/`AllowClause`/`Severity`/`ScopePredicate` types and `Compile`'s happy path —
> so test 1 compiles. Then the fail-closed error cases (2–6). Then the loader + scenario 21 + the
> archguard guard (7–10). `closure.Safe` is **not** modified.

## Slice 1 — `scope.Compile` (package scope, no loader)
1. [ ] `Compile` lowers a TaskContract with a `resource` clause and a `selector` clause to a
   `ScopePredicate` whose `Clauses` equal the expected `closure.ScopeClause`s (as built by
   `closure.ResourceClause` / `closure.SelectorClause`) and whose `MaxSeverity` is carried through.
2. [ ] `Compile` returns an error for an unsupported/unknown dimension (`dim: ownership`) — fail
   closed, not a silent skip.
3. [ ] `Compile` returns an error for a wrong or absent `apiVersion` or `kind` (not `krsm.io/v1` /
   `TaskContract`).
4. [ ] `Compile` returns an error for a `maxSeverity` outside the known set.
5. [ ] `Compile` of an empty `spec.allow` yields an empty `ScopePredicate` (authorises nothing) with
   **no** error.
6. [ ] `Compile` returns an error when an allow-clause is structurally invalid (e.g. a `selector`
   clause carrying a `name`) — `closure.ScopeClause.Validate` surfaced through the compiler.

## Slice 2 — loader, golden scenario, archguard
7. [ ] `scenario.Load` on a directory containing `taskcontract.yaml` compiles it and produces a
   `Scenario` whose `closure.Safe` verdict is `Allow` with the expected three-ref closure
   (scenario `21-taskcontract-selector-scope`) — the TaskContract path classifies correctly
   end-to-end.
8. [ ] `scenario.Load` on a directory with only `scope.yaml` still loads via the legacy path (a
   `scope.yaml`-only scenario still classifies) — back-compat preserved.
9. [ ] `archguard`'s `TestScopeIsStdlibOnly` passes: `github.com/ridik-il/krsm/scope/...` depends
   only on the standard library plus the `github.com/ridik-il/krsm` module.
10. [ ] **Regression:** all 20 existing golden scenarios still produce their expected `Decision`
    (the `scope.yaml` path and `closure.Safe` are unchanged).

## Non-test deliverables (build phase)
- `CHANGELOG.md` entry: new public `scope` package + `TaskContract`→`ScopePredicate` compiler.
- `docs/adr/0009-scope-package-and-taskcontract-compiler.md`: the decision (realising ADR-0003);
  scope/ embeddable & stdlib-only; `Compile` takes a struct not YAML; fail-closed on a bad contract.
- `make check` green (gofmt, go vet, golangci-lint, staticcheck, `go test -race ./...`); both
  `archguard` invariants green (`closure` and `scope` stdlib-only).
