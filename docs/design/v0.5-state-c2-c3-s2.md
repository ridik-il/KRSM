# Design: v0.5 state path — indexed informer State + staleness guard (C2 + C3) + CLI group-aware target (S2)
Status: approved · Date: 2026-06-27 · Slug: `v0.5-state-c2-c3-s2` · Plan: docs/plans/v0.5-state-c2-c3-s2.md
Issue: #21 (slice 3) + #22 (slice 4) · resolves #14 (C3), #13 (C2), #16 (S2)

## Summary
A new **`state/` package** provides a second `closure.State` implementation (beside
`closure.NewScanState`): an informer-backed, incrementally **indexed** `Provider` that
serves the nine `State` methods from four inverted indexes in `O(c·d)` (blast radius), not
`O(c·n)` (cluster size). Secrets and ConfigMaps are watched **metadata-only**
(`PartialObjectMetadata`) so their `data` is never fetched or cached (C3, #14). A single
watch-fed cache replaces the v0.4 non-atomic multi-`List` (C2-part, #13); a per-request
**staleness guard** (`FreshGet` + `resourceVersion` drift check, fail-closed on unresolved
drift) bounds the residual watch-vs-write race (C2, #13). Separately, the CLI gains a
**group-aware** target (`Kind.group/name`) with fail-closed disambiguation (S2, #16). The
`Provider` reuses the slice-2 index-free `project` so it reproduces goldens 01–27 verbatim
(the corpus parity oracle); `closure`/`scope`/`Ref.human`/the goldens are **unchanged**.
Built strictly test-first (`design-check-tdd → tdd`).

## Interfaces / contracts

### `state/` package — the indexed `closure.State` (slice 3)
```go
package state

import (
    "context"
    "k8s.io/client-go/rest"
    "github.com/ridik-il/krsm/closure"
)

// Options carries what the Provider needs to build its informers. trackedGVRs come from
// the slice-1 hardened discovery selection (preferred-version, C1-tolerant); metadataGVRs
// is the subset watched metadata-only (Secret, ConfigMap). resync is the informer resync
// period (0 = no periodic resync; watch deltas only).
type Options struct {
    Resync time.Duration // default 0
    // tracked/metadata GVR sets are resolved internally from discovery (NewFromConfig),
    // or injected for tests via newProvider.
}

// Provider is the informer-backed, incrementally indexed closure.State (ADR-0004,
// DESIGN §4). Read-only: it runs only get/list/watch over the tracked GVRs and NEVER a
// write verb. Secrets/ConfigMaps are PartialObjectMetadata, so their data is never cached.
type Provider struct {
    mu       sync.RWMutex
    byUID    map[string]*closure.Object       // uid     → object  (Get fast path)
    byHuman  map[string]*closure.Object       // Kind/ns/name → object (Get fallback)
    owner    map[string][]closure.Ref         // owner uid → child refs
    ns       map[string][]closure.Ref         // namespace → contained refs (excl. Namespace)
    selOwner map[string][]closure.Ref         // namespace → Service/PDB/NetworkPolicy refs
    xref     map[string][]closure.Ref         // referentKey → consumer refs (any cross-ref)
    factories []sharedFactory                  // full + metadata informer factories
    getter    objectGetter                     // dynamic + metadata clients for FreshGet
    scope     cluster.ScopeInfo                // discovered GVK → namespaced
    synced    func() bool
}

var _ closure.State = (*Provider)(nil)

// New builds a Provider from a *rest.Config: resolves tracked + metadata GVR sets and the
// ScopeInfo via the slice-1 C1-tolerant discovery, constructs the full and metadata
// informer factories, and wires the four indexes' event handlers. It does NOT block.
func New(cfg *rest.Config, opts Options) (*Provider, error)

// Start launches every informer (full + metadata). Non-blocking.
func (p *Provider) Start(ctx context.Context)

// WaitForCacheSync blocks until every tracked informer's cache is initially populated.
// The caller MUST NOT serve a verdict before this returns true — an unsynced cache is an
// unknown closure (fail-closed, DESIGN §5). Returns false if ctx is cancelled first.
func (p *Provider) WaitForCacheSync(ctx context.Context) bool

// The nine closure.State methods, each served from the indexes under the read lock,
// returning copied []closure.Ref (callers must not see the backing slice):
func (p *Provider) Get(r closure.Ref) (*closure.Object, bool)
func (p *Provider) OwnedChildren(r closure.Ref) []closure.Ref
func (p *Provider) NamespaceContents(ns string) []closure.Ref
func (p *Provider) PodsSelectedBy(r closure.Ref) []closure.Ref
func (p *Provider) PodsMatching(ns string, sel closure.LabelSelector, ownerKind string) []closure.Ref
func (p *Provider) SelectorsTargeting(pod closure.Ref) []closure.Ref
func (p *Provider) SelectorsMatchingLabels(ns string, labels map[string]string) []closure.Ref
func (p *Provider) Consumers(target closure.Ref) []closure.Ref
func (p *Provider) ControllersTargeting(r closure.Ref) []closure.Ref

// FreshGet does a bounded, on-demand live GET for ONE ref (the staleness guard, slice 4).
// It is metadata-typed for Secret/ConfigMap (never pulls data) and a dynamic GET
// otherwise, projected through the SAME project() the informers use. Never a re-list.
func (p *Provider) FreshGet(ctx context.Context, ref closure.Ref) (closure.Object, bool, error)
```

The nine methods reproduce `scanState`'s semantics **exactly** (closure/state.go is the
oracle), only served from indexes instead of a linear scan:

| Method | scanState behaviour to match | Index path |
|---|---|---|
| `Get` | uid then Kind/ns/name (`lookup`) | `byUID[uid]` else `byHuman[human]` |
| `OwnedChildren` | children whose `Owners[].UID == owner.UID` | `owner[owner.UID]` |
| `NamespaceContents` | namespaced objs in ns, **excluding Kind=Namespace** | `ns[namespace]` (Namespaces never indexed here) |
| `PodsSelectedBy` | `PodsMatching(owner.ns, owner.Selector, owner.Kind)` | `lookup` owner → `PodsMatching` |
| `PodsMatching` | pods in ns where `selectorBinds(ownerKind, sel, labels)` | scan ns pod bucket; reuse `selectorBinds` |
| `SelectorsTargeting` | `SelectorsMatchingLabels(pod.ns, pod.Labels)` | `lookup` pod → `SelectorsMatchingLabels` |
| `SelectorsMatchingLabels` | Service/PDB/NetworkPolicy in ns binding labels | `selOwner[ns]` filtered by `selectorBinds` |
| `Consumers` | cross-refs **excluding `RefScaleTarget`**, via `crossRefMatches` | `xref[referentKey(target)]` filtered ≠ scaleTarget |
| `ControllersTargeting` | **only `RefScaleTarget`** cross-refs | `xref[referentKey(target)]` filtered = scaleTarget |

`selectorBinds` (empty-selector kind-awareness: empty Service selector binds nothing,
empty NetworkPolicy/PDB/workload selector binds all ns pods, nil binds nothing) and
`crossRefMatches` (uid when both present, else Kind/ns/name) live in `closure` and are
reused — the `Provider` must not re-derive selector or cross-ref matching.

### CLI group-aware target (S2, #16) — `cmd/krsm` + `internal/cluster/reader.go`
```go
// splitTarget now accepts an optional ".group" on the Kind: "<Kind>[.<group>]/<name>".
// Returns the bare Kind token, the group ("" if unqualified), and the name.
func splitTarget(target string) (kind, group, name string, err error)

// ResolveKind becomes group-aware. With group != "", it resolves the resource in THAT
// group (error if that group serves no such Kind). With group == "" and the Kind served
// by exactly one group, it resolves as before. With group == "" and the Kind served by
// MORE THAN ONE group, it returns a fail-closed error listing the candidate groups —
// never a silent first-match.
func (r *Reader) ResolveKind(ctx context.Context, kind, group string) (canonicalKind string, err error)
```
`Ref.human` stays group-agnostic; the resolved target `Ref` carries the canonical Kind as
today. The webhook's uid-based match (slice 5) is the complete S2 fix; the CLI fix removes
the *silent* wrong-target footgun and lets an operator disambiguate.

## Data model
- **No new persisted/wire types.** The indexes store only `closure.Object` projections
  (the per-relation fields), never raw API objects, so Secret/ConfigMap `data` is
  structurally absent from the cache.
- **`referentKey(ref)`** — the cross-ref index key: `Kind/namespace/name` (group-agnostic,
  matching `crossRefMatches`'s name path; see parity note). Consumers/ControllersTargeting
  look up by the *target*'s `referentKey`.
- **`Options`** adds `Resync time.Duration`. CLI S2 adds a `group` field threaded through
  `splitTarget`→`ResolveKind`; no struct persisted.

## Behavior

### Index maintenance (event handlers, slice 3)
Each informer add/update/delete handler `project()`s the object, then under a single
`sync.RWMutex` write lock updates every index and both lookup maps:
- **add:** insert into `byUID`/`byHuman`; append the object's `Ref` to `ns[namespace]`
  (unless Kind=Namespace); for each `Owners[].UID`, append the child to `owner[uid]`; if a
  selector kind, append to `selOwner[ns]`; for each `CrossRefs[]`, append the consumer to
  `xref[referentKey(crossref.Ref)]`.
- **update:** delete-then-add by uid (recompute every edge; an object's labels/owners/refs
  can change). resourceVersion stored on the cached `Object` for the staleness check.
- **delete:** remove the object and every edge it contributed (its uid bucket, its ns
  entry, its owner-child entries, its selector entry, its xref consumer entries).

Reads take the **read lock** and return a **copied** `[]closure.Ref` (never the backing
slice). Concurrency correctness is pinned by `go test -race`.

### Parity-critical: empty cross-ref uids (the central correctness point)
The index consumes the slice-2 **index-free `project`**, which leaves each cross-ref's
referent `UID` empty (a live informer sees objects one at a time — there is no whole-set
moment, so `BuildObjects`'s `resolveCrossRefUIDs` post-pass has no equivalent here). The
engine's `crossRefMatches` matches by uid only when **both** sides carry one, else by
Kind/ns/name; with empty referent uids it takes the Kind/ns/name path, which selects the
same object. Therefore the closure/escaping set is **identical** to `NewScanState`'s —
this is exactly slice 2's established guarantee, now relied on live. The corpus parity
oracle (below) proves it; `referentKey` is correspondingly Kind/ns/name-based.

### Startup / cache-sync gate (fail-closed)
`New` resolves GVRs + scope via the slice-1 C1-tolerant discovery (a flaky aggregated
group must not crash `Start`). `Start` launches informers; `WaitForCacheSync` must return
true before the caller serves. Until synced, the `Provider` reports not-ready via `synced`;
the caller (the future webhook) maps that to a fail-closed deny "state not ready". This
branch's tests assert the not-synced state is observable and distinct (never a silent empty
closure).

### Staleness guard (slice 4, C2)
The hot path is index-only. Given the authoritative request object's `resourceVersion`
(supplied by the webhook in slice 5; supplied by tests here), the guard compares it to the
cached copy. **In sync → no fetch.** **Request newer than cache (drift) →** a bounded
`FreshGet` over the `O(d)` closure neighbourhood (the target + its closure members), never
a re-list. **Unresolved drift** (FreshGet still can't reconcile) → **fail closed** with the
distinct, credential-free reason **"could not confirm current state"**. `generation` is a
secondary spec-change hint; `resourceVersion` is the primary signal. The webhook is the
real consumer; this branch builds and unit-tests the mechanism hermetically.

### CLI S2 flow
`runCheckLive` → `splitTarget` yields `(kind, group, name)` → `ResolveKind(ctx, kind,
group)`. Qualified group resolves that group; unqualified + unique resolves as today;
unqualified + ambiguous → fail-closed error listing candidates (exit 1). Unchanged
otherwise.

### Fail-closed taxonomy (distinct, credential-free reasons)
- Cache not synced → "state not ready".
- Staleness drift unresolved → "could not confirm current state".
- Discovery/list/get error, internal error → "cluster read failed, denying fail-closed: …"
  (wraps only the API error; never the `rest.Config`, tokens, or object `data`).
- Ambiguous bare Kind (CLI) → an operator-facing disambiguation error listing groups.

## Components & responsibilities
- `state/provider.go` — `Provider`, `New`/`newProvider` (injection seam), `Start`,
  `WaitForCacheSync`, the nine `State` methods, `FreshGet`. Read-only.
- `state/index.go` — the four indexes + the byUID/byHuman maps, the add/update/delete
  handler logic, `referentKey`. Consumes `internal/cluster.project`.
- `state/informers.go` — full `SharedInformerFactory` + `metadatainformer.SharedInformerFactory`
  split; one informer per preferred-version GVR; reuses slice-1
  `serverPreferredResources`/`selectTargets`/`scopeFromLists` + C1 tolerance.
- `state/freshget.go` — metadata-typed vs dynamic GET, projected via `project`.
- `state/source_guard_test.go` — forbids any write verb in `state/` (mirrors
  `internal/cluster.TestPackageInvokesNoWriteVerbs`).
- `internal/cluster/reader.go` — `ResolveKind` gains the `group` parameter + ambiguity
  fail-close. `internal/cluster/build.go` — `project` reused (unchanged).
- `cmd/krsm/main.go` — `splitTarget` parses `Kind.group/name`; `runCheckLive` threads
  `group`.
- `internal/archguard` — confirm `state/` deps don't leak into `closure`/`scope`.

## Alternatives considered
- **One whole-cluster informer relist per request (reuse the one-shot reader).** Rejected:
  `O(n)` per admission, re-triggers C2/C3 every action — DESIGN §4/ADR-0004 chose informers.
- **Full informers for Secrets/ConfigMaps.** Rejected (C3): caches every Secret's `data` —
  an exposure surface the four relations never read. `PartialObjectMetadata` carries
  labels/owners/finalizers, all that's needed.
- **Resolve cross-ref uids in the index at event time.** Rejected (slice-2 decision): the
  Kind/ns/name fallback already gives the correct match; uid plumbing buys nothing and
  complicates delete handling.
- **Always-GET the target each request (no drift check).** Rejected (ADR-0004 Q3): defeats
  the index's read-free hot path; drift-triggered `FreshGet` keeps the common case fast.
- **Change `Ref.human` to include the group (true S2 fix in the engine).** Rejected: ripples
  into goldens 01–27 and `crossRefMatches`; out of scope. The webhook's uid match is the
  complete fix; the CLI gets a fail-closed disambiguation now.
- **Per-index mutex.** Rejected for v0.5: one `RWMutex` is simpler and correct; reads are
  short slice-copies. Revisit only if contention shows (single-replica, low QPS).

## Trade-offs & risks
- **Index/loader parity (highest).** The index must reproduce `project`'s four relations and
  rely on the Kind/ns/name cross-ref fallback. Mitigation: reuse `project` verbatim; the
  corpus parity oracle gates it before any cluster.
- **Staleness window (ADR-0004).** Bounded, not eliminated (K8s has no multi-resource
  snapshot). Mitigation: drift-triggered `FreshGet` + fail-closed; the real request object
  arrives with the webhook (slice 5) — this branch tests the mechanism.
- **Metadata/full split correctness.** A selector-relevant kind fetched metadata-only would
  silently break binding. `PartialObjectMetadata` carries labels, so Secrets/ConfigMaps
  (referenced-only) are safe; Pods stay full. Pinned by a metadata-Pod-binds-Service test.
- **Informer startup on a missing group.** Reuse C1 tolerance so a missing/flaky aggregated
  group doesn't crash `Start`.
- **`-race`.** Concurrent handler writes vs method reads — single `RWMutex` + copied returns
  must be `-race` clean.
- **S2 residual.** Two same-Kind objects in different groups sharing ns/name still share a
  human key in the engine; acceptable (CLI resolves the right GVR; webhook uid is the full
  fix); documented.

## Security (DESIGN §9, securing-kubernetes-on-cloud / testing-for-sensitive-data-exposure)
- **No Secret data at rest in memory.** Secret/ConfigMap via `PartialObjectMetadata`; the
  cache stores `closure.Object` projections only. Pinned: a cached Secret carries
  labels/owners/finalizers but no `data`.
- **Least privilege, read-only.** Only get/list/watch over the tracked GVRs; no write, no
  wildcard. Enforced by the `state/` source-guard test. (RBAC *manifest* is slice 7; here
  it's the documented contract.)
- **Fail-closed, credential-free reasons.** The taxonomy above; no reason/log line echoes
  `rest.Config`, bearer tokens, client certs, or object `data`.

## Testability
Hermetic-first: fake clientset (full informers), fake metadata client (metadata informers),
fake dynamic (FreshGet), crafted from the existing corpus fixtures. The ordered test list
below is the `design-check-tdd` input; each test maps to a plan "Wanted behaviour" seed.

**Slice 3 — indexed `state.Provider`:**
1. `TestProviderImplementsState` — `var _ closure.State = (*Provider)(nil)` (compile guard).
2. `TestProviderParityOracle` — feed corpus objects 01–27 through the informer index; the
   `closure.Safe` verdict + closure/escaping/external set equals `NewScanState`'s for each.
3. `TestProviderMethodsMatchScanState` — table over the corpus: `Get`, `OwnedChildren`,
   `NamespaceContents`, `PodsSelectedBy`, `PodsMatching`, `SelectorsTargeting`,
   `SelectorsMatchingLabels`, `Consumers`, `ControllersTargeting` each return the **same Ref
   set** (order-insensitive) as `NewScanState`.
4. `TestConsumersExcludesScaleTarget` / `TestControllersTargetingOnlyScaleTarget` — the
   `RefScaleTarget` split matches scanState.
5. `TestNamespaceContentsExcludesNamespaceKind` — a Namespace object is not its own content.
6. `TestEmptySelectorKindAwareness` — empty Service selector binds nothing; empty
   NetworkPolicy/PDB/workload selector binds all ns pods (corpus #8).
7. `TestSecretCachedMetadataOnly` — a cached Secret carries labels/owners/finalizers, **no
   `data`**; same for ConfigMap.
8. `TestMetadataPodBindsServiceSelector` — a `PartialObjectMetadata` Pod (labels present)
   is still returned by `SelectorsTargeting`/`PodsSelectedBy`.
9. `TestMultiVersionInformedOnce` — a two-version resource yields one informer (preferred).
10. `TestServeBeforeSyncFailsClosed` — before `WaitForCacheSync` returns true, the
    not-synced state is observable/distinct (never a silent empty closure).
11. `TestUpdateAndDeleteMaintainIndexes` — add→update(labels/owners/refs change)→delete
    leaves every index and both lookup maps consistent with a fresh build.
12. `TestStateNoWriteVerb` — source-guard: no write verb in `state/`.
13. `-race`: `TestConcurrentReadsDuringHandlerWrites`.

**Slice 4 — staleness guard:**
14. `TestDriftTriggersBoundedFreshGet` — request `resourceVersion` newer than cache → a
    bounded `FreshGet` over the neighbourhood; **no `list` action** observed.
15. `TestInSyncNoFreshGet` — cache matches request rv → no `FreshGet` (hot path read-free).
16. `TestUnresolvedDriftFailsClosed` — drift FreshGet can't reconcile → fail closed with
    "could not confirm current state".
17. `TestFreshGetMetadataKindNoData` — `FreshGet` for a Secret/ConfigMap uses the metadata
    client and returns no `data`.
18. `TestStalenessReasonCredentialFree` — the drift reason/log never echoes config/token/data.

**S2 — CLI group-aware target:**
19. `TestSplitTargetParsesGroupQualifier` — `Deployment.apps/web` → (`Deployment`,`apps`,`web`).
20. `TestResolveKindQualifiedGroupResolves` — qualified group resolves that group's resource.
21. `TestResolveKindAmbiguousBareKindFailsClosed` — a Kind in >1 group, unqualified → error
    listing the candidate groups (no first-match guess).
22. `TestResolveKindUnambiguousBareKindUnchanged` — single-group bare Kind resolves as today
    (existing CLI tests stay green).
23. `TestResolveKindWrongGroupFailsClosed` — `Kind.group` whose group serves no such Kind →
    fail-closed error.

Regression: full `make check` — corpus parity 01–27, `internal/cluster` read-only guards,
archguard (`closure`/`scope` stdlib-only; `state/` outside), `go test -race`.

## Open questions
None blocking — the five plan open questions were resolved at approval (one branch;
`project` in `internal/cluster`; metadata-typed `FreshGet`; `resourceVersion` primary;
full informers for CRDs). Design sub-decisions taken here (single `RWMutex`; `referentKey`
= Kind/ns/name; delete-then-add on update) are recorded above and revisitable in `tdd` if a
test exposes a gap.
